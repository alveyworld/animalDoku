## 1. Problem Statement

UI components need shared colors, spacing, and typography to match the GDD's soft pastel aesthetic. Ad-hoc magic numbers in views cause inconsistent UI and harder accessibility work later.

## 2. Goals

- Define pastel color palette and semantic color names.
- Define spacing scale and corner radius constants.
- Define minimum touch target size (44pt).
- Define high-contrast color variant tokens (used in P5.3).

## 3. Non-Goals

- Full component library.
- Dark mode theming beyond high-contrast toggle.

## 4. Personas & User Stories

- **As a player**, I want a calm, consistent visual style across screens.
- **As a developer**, I want design tokens so that UI code uses named constants.
- **As a player using accessibility settings**, I want tokens that support high contrast (wired in P5.3).

## 5. Functional Requirements

- **FR-1.** Colors MUST include: background, surface, primary, accent, error/violation, region palette helpers.
- **FR-2.** `TouchTarget.minimum` MUST be ≥ 44pt.
- **FR-3.** Corner radius and padding constants MUST be centralized in `UI/DesignSystem/`.
- **FR-4.** High contrast variants MUST be defined alongside default tokens.

## 6. Non-Functional Requirements

- **Accessibility** — Token names document intended contrast usage.
- **Maintainability** — Swift extensions or enum namespace, no scattered literals in views.

## 7. Acceptance Criteria

- **AC-1.** *Given* `AppColors.background`, *when* used in a preview, *then* a pastel tone renders.
- **AC-2.** *Given* `TouchTarget.minimum`, *when* read, *then* value is ≥ 44.
- **AC-3.** *Given* high contrast tokens, *when* compared to default, *then* border/contrast values are stronger.

## 8. Data Model
