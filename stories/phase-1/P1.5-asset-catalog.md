## 1. Problem Statement

Theme icons, sounds, and app branding need a organized asset location before art is finalized. Setting up the catalog early avoids reorganizing resources mid-project.

## 2. Goals

- Configure `Assets.xcassets` with logical namespaces.
- Add placeholder groups: AppIcon (empty slot), Themes/Frogs, Themes/Dogs, Themes/Foxes.
- Add `Resources/Sounds/` folder reference in project.

## 3. Non-Goals

- Final artwork or app icon (P5.4).
- Theme-specific sounds (P4.5).

## 4. Personas & User Stories

- **As a developer**, I want asset namespaces ready so that I can drop in icons without restructuring.
- **As a designer** (future), I want clear slots for three MVP themes.

## 5. Functional Requirements

- **FR-1.** Asset catalog MUST exist and be included in app target.
- **FR-2.** Theme image sets MUST be named `theme-frogs-icon`, `theme-dogs-icon`, `theme-foxes-icon` (or equivalent convention documented in catalog).
- **FR-3.** `Resources/Sounds/` folder MUST exist in project (may be empty).

## 6. Non-Functional Requirements

- **Maintainability** — Document naming convention in catalog README comment or P4.1 story.

## 7. Acceptance Criteria

- **AC-1.** *Given* build, *when* run, *then* missing images do not crash (use SF Symbol fallbacks in code until P4.1).
- **AC-2.** *Given* Assets.xcassets, *when* inspected, *then* theme groups exist.
