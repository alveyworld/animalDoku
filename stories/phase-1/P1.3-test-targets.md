## 1. Problem Statement

Game logic correctness depends on validator and session tests. Without test targets configured early, tests are added late and regressions slip through.

## 2. Goals

- Add `AnimalDokuTests` unit test target.
- Add `AnimalDokuUITests` UI test target.
- Include one passing sample test in each target.
- Wire test targets to main app target.

## 3. Non-Goals

- Full test coverage (comes in P2.9, P3.7, P6.x).
- CI integration.

## 4. Personas & User Stories

- **As a developer**, I want unit tests runnable via Cmd+U so that I can TDD the engine.
- **As a developer**, I want a UI test target so that I can automate play-to-win later.

## 5. Functional Requirements

- **FR-1.** Unit test target MUST compile and link against app module (`@testable import AnimalDoku`).
- **FR-2.** UI test target MUST launch the app in simulator.
- **FR-3.** Each target MUST contain at least one passing placeholder test.

## 6. Non-Functional Requirements

- **Reliability** — Tests MUST be deterministic (no network, no clock dependency in placeholders).

## 7. Acceptance Criteria

- **AC-1.** *Given* Xcode Test action, *when* run, *then* all placeholder tests pass.
- **AC-2.** *Given* `AnimalDokuTests`, *when* a new test file is added under `Tests/`, *then* it is discovered by Xcode.
