## 1. Problem Statement

The repository has no iOS application target. Without an Xcode project, no Swift code can compile, run on simulator, or ship to TestFlight. This is the foundation for all development.

## 2. Goals

- Create a SwiftUI iOS app that builds and runs on simulator.
- Target iOS 17+ per GDD.
- Enable portrait-only orientation for MVP.
- Establish bundle identifier and app display name.

## 3. Non-Goals

- Game logic, UI screens, or assets beyond defaults.
- App Store Connect setup.
- CI/CD pipeline.

## 4. Personas & User Stories

- **As a developer**, I want a compilable Xcode project so that I can implement features incrementally.
- **As a developer**, I want iOS 17+ and SwiftUI configured so that I can use `@Observable` and modern APIs.
- **As a player** (indirect), I want the app to launch to a blank shell without crashing.

## 5. Functional Requirements

- **FR-1.** The project MUST be an iOS app using SwiftUI lifecycle (`@main` App struct).
- **FR-2.** The deployment target MUST be iOS 17.0 or later.
- **FR-3.** The app MUST support iPhone in portrait orientation only for MVP.
- **FR-4.** The project MUST build and run on iOS Simulator without errors.

## 6. Non-Functional Requirements

- **Performance** — N/A for empty shell.
- **Security** — No network, keys, or entitlements beyond defaults.
- **Privacy** — No data collection in this story.
- **Accessibility** — N/A.
- **Maintainability** — Use standard Xcode project layout; name target `AnimalDoku`.

## 7. Acceptance Criteria

- **AC-1.** *Given* a clean clone, *when* `xcodebuild` or Xcode Run is executed, *then* the app launches on simulator.
- **AC-2.** *Given* the project settings, *when* inspected, *then* deployment target is iOS 17+ and interface is SwiftUI.
- **AC-3.** *Given* device rotation, *when* on iPhone, *then* app remains portrait-locked.
