## 1. Problem Statement

Without a consistent folder structure, code will scatter across the Xcode project and create coupling between engine logic and UI. A clear module layout reduces rework as features are added.

## 2. Goals

- Create physical groups matching the architecture in the roadmap.
- Establish separation: Models, Engine (no SwiftUI), Services, ViewModels, UI.
- Add empty placeholder files or groups where needed for Xcode navigation.

## 3. Non-Goals

- Implementing any business logic.
- Swift Package Manager modules (single target is fine for MVP).

## 4. Personas & User Stories

- **As a developer**, I want predictable file locations so that I know where to add new code.
- **As a developer**, I want Engine isolated from UI so that I can unit test without SwiftUI.

## 5. Functional Requirements

- **FR-1.** The project MUST contain groups: `App/`, `Models/`, `Engine/`, `Services/`, `ViewModels/`, `UI/`, `Themes/`, `Resources/`.
- **FR-2.** `UI/` MUST contain `DesignSystem/`, `Components/`, `Screens/` subfolders.
- **FR-3.** `Resources/` MUST contain `Puzzles/` and `Sounds/` subfolders.
- **FR-4.** `Tests/` MUST mirror `Engine/` and `Models/` for unit tests.

## 6. Non-Functional Requirements

- **Maintainability** — Match [Roadmap §5](../../IMPLEMENTATION_ROADMAP.md#5-suggested-architecture) exactly.
- **Backward compatibility** — N/A (greenfield).

## 7. Acceptance Criteria

- **AC-1.** *Given* the Xcode project navigator, *when* expanded, *then* all roadmap folders exist as groups.
- **AC-2.** *Given* a new `Validator.swift`, *when* added, *then* it lives under `Engine/` not `UI/`.
- **AC-3.** *Given* the project, *when* built, *then* no compile errors from empty groups.
