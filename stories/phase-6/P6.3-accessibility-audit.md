## 1. Problem Statement

The GDD commits to a concrete accessibility contract — large touch targets, colorblind-friendly regions, high contrast mode, sound toggle, Reduce Motion, and VoiceOver announcements per cell. These requirements are spread across many stories and easy to implement partially. Without a dedicated audit, the app can ship with unlabeled cells, sub-44pt targets, or contrast failures that block or degrade play for disabled users and jeopardize App Store review. This story is the consolidated gate that verifies the whole GDD accessibility table is met before release.

## 2. Goals

- Verify every item in the [GDD Accessibility table](../../AnimalDoku_Game_Design_Document.md#accessibility) is implemented and testable.
- Confirm VoiceOver announces each cell's row, column, region, and state, and that all controls have labels and traits.
- Confirm Dynamic Type scales UI text without truncation or overlap up to the largest accessibility sizes.
- Confirm color/contrast meets ≥4.5:1 for text and icons (default and High Contrast mode) and that regions are distinguishable without color (pattern/border).
- Confirm Reduce Motion disables non-essential animations and the sound toggle silences all SFX.
- Produce a written audit checklist with pass/fail evidence to attach to the release.

## 3. Non-Goals

- Implementing the underlying features (owned by P3.x and P5.3) — this story verifies and files defects, plus small labeling fixes.
- Automated UI play-to-win flow (P6.2) — though this audit relies on the identifiers/labels it adds.
- Device matrix/layout testing across many screen sizes (P6.4).
- Localization/i18n correctness beyond string externalization checks.

## 4. Personas & User Stories

- **As a VoiceOver user**, I want each cell to announce its position, region, and state so that I can solve the puzzle without sight.
- **As a low-vision user**, I want High Contrast mode and large text so that I can read the board comfortably.
- **As a colorblind user**, I want regions distinguishable by pattern/border so that color is not the only cue.
- **As a motion-sensitive user**, I want Reduce Motion respected so that animations do not cause discomfort.
- **As a reviewer**, I want documented a11y conformance so that App Store review passes.

## 5. Functional Requirements

- **FR-1.** Every interactive cell and toolbar button MUST have a touch target ≥44×44 pt (GDD large touch targets).
- **FR-2.** Each board cell MUST expose a VoiceOver label announcing row, column, region, and current state (empty / blocked / animal), plus appropriate traits.
- **FR-3.** Each control (undo, redo, reset, hint, mode toggle, settings, Play Again) MUST have a VoiceOver label and value/trait; disabled controls MUST announce as disabled.
- **FR-4.** All user-facing text MUST scale with Dynamic Type up to the largest accessibility size without clipping, truncation, or overlap.
- **FR-5.** Text and icon contrast MUST meet ≥4.5:1 in default mode and in High Contrast mode (P5.3) across all three themes.
- **FR-6.** Regions MUST be distinguishable without relying on color alone (pattern or border), verified in a grayscale simulation.
- **FR-7.** With Reduce Motion enabled, placement/win animations MUST degrade to non-animated or minimal transitions (per P5.1/P5.2).
- **FR-8.** The sound toggle MUST silence all game SFX when off (per P4.5).
- **FR-9.** VoiceOver focus order MUST be logical (board row-major, then controls) with no traps.
- **FR-10.** An automated accessibility snapshot check (XCTest `performAccessibilityAudit()` on key screens) SHOULD run in CI and report zero critical issues.

## 6. Non-Functional Requirements

- **Performance** — Audit tooling adds negligible runtime; `performAccessibilityAudit` runs in the UI test target.
- **Security** — N/A.
- **Privacy** — N/A.
- **Accessibility** — This story **is** the accessibility gate; target WCAG 2.1 AA-equivalent for contrast (≥4.5:1) and Apple accessibility guidance.
- **Maintainability** — Labels centralized via view modifiers/helpers; contrast token pairs documented alongside design tokens (P1.4).
- **Internationalization** — Confirm labels are localizable string keys, not hard-coded literals.
- **Observability** — Audit results recorded as a checklist artifact in the repo/release notes.

## 7. Acceptance Criteria

- **AC-1.** *Given* VoiceOver on, *when* focusing any cell, *then* it announces row, column, region, and state (e.g. "Row 3, Column 5, blue region, empty").
- **AC-2.** *Given* VoiceOver on, *when* traversing controls, *then* each has a meaningful label and disabled controls are announced as dimmed/disabled.
- **AC-3.** *Given* the largest Dynamic Type accessibility size, *when* viewing all screens, *then* no text is clipped, truncated, or overlapping.
- **AC-4.** *Given* each theme in default and High Contrast mode, *when* contrast is measured, *then* text/icon ratios are ≥4.5:1.
- **AC-5.** *Given* a grayscale simulation, *when* viewing the board, *then* all regions remain visually distinguishable.
- **AC-6.** *Given* Reduce Motion on, *when* placing an animal and winning, *then* animations are disabled or minimal.
- **AC-7.** *Given* the sound toggle off, *when* performing any action, *then* no SFX play.
- **AC-8.** *Given* all interactive elements, *when* measured, *then* every target is ≥44×44 pt.
- **AC-9.** *Given* `performAccessibilityAudit()` on Game, Settings, and Win screens, *when* run, *then* zero critical issues are reported (or each is triaged with justification).

## 8. Data Model

No new production types. Deliverable audit artifact:

```text
docs/accessibility-audit.md  (checklist with pass/fail + evidence)
| Requirement           | Method                     | Result |
|-----------------------|----------------------------|--------|
| ≥44pt targets         | Accessibility Inspector    | PASS   |
| Cell VoiceOver labels | Manual VoiceOver pass      | PASS   |
| Dynamic Type          | Largest AX size on device  | PASS   |
| Contrast ≥4.5:1       | Color Contrast Calculator  | PASS   |
| Colorblind regions    | Grayscale simulation       | PASS   |
| Reduce Motion         | System setting on          | PASS   |
| Sound toggle          | Manual                     | PASS   |
```

## 9. API Surface

- No network API. Optional automated audit in the UI test target:

```swift
func testGameScreenAccessibility() throws {
    let app = XCUIApplication(); app.launch()
    try app.performAccessibilityAudit()   // XCTest, iOS 17+
}
```

## 10. UI / UX

- May add/adjust `.accessibilityLabel`, `.accessibilityValue`, `.accessibilityHint`, and `.accessibilityElement(children:)` on `CellView`, `BoardView`, toolbar, settings, and win overlay.
- Verify focus order and grouping; ensure the board reads as a grid, not a flat list of 64 undifferentiated cells.
- Confirm empty/locked (post-win) states are announced.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Cell announcements verbose/annoying | M | M | Concise label format; use value for state; user-test wording |
| Contrast fails in one theme | M | M | Adjust theme token pair (coord with P4.1/P5.3) before release |
| Dynamic Type breaks board layout | M | M | Cap board cell text scaling; scale chrome/labels, keep grid fixed |
| Automated audit yields false positives | M | L | Triage each flag; document intentional exceptions |

## 15. Rollout Plan

- No feature flag. Merge when AC-1–AC-9 pass and the audit artifact is complete.
- CI: add `AccessibilityTests` to required checks once stable.
- Rollback: labeling changes are low-risk; revert individual fixes if regressions appear.

## 16. Test Plan

- **Automated** — `performAccessibilityAudit()` on Game, Settings, Win screens (AC-9); target ≥44pt via inspector.
- **Screen reader** — Scripted VoiceOver pass: launch → focus cells → place via VoiceOver → win → Play Again (AC-1, AC-2).
- **Dynamic Type** — Largest AX size across all screens (AC-3).
- **Contrast** — Measure each theme × (default, high contrast) with a contrast tool (AC-4).
- **Colorblind** — Grayscale + Sim Daltonism check of regions (AC-5).
- **Motion / sound** — Reduce Motion on (AC-6); sound toggle off (AC-7).
- **Manual exploratory** — QA checklist filled into `docs/accessibility-audit.md`.

## 17. Documentation & Training

- Publish `docs/accessibility-audit.md` as the living conformance record.
- Add an "Accessibility checklist" to the PR template so new UI is labeled by default.
