## 1. Problem Statement

The final step of shipping Animal Doku is a complete App Store Connect listing and a submission that passes Apple review. This requires marketing metadata (name, subtitle, description, keywords), screenshots at required device sizes, age rating, category, pricing, the privacy label, and a review-ready build. Missing or inaccurate metadata is the most common cause of rejection or delay. This story assembles the full listing and drives the release candidate through review to public availability.

## 2. Goals

- Complete the App Store Connect listing: name, subtitle, description, keywords, category, support/marketing URLs.
- Provide required screenshots (and optional preview) for the supported device sizes.
- Set age rating, pricing (free), availability, and confirm the privacy "nutrition label" (from P7.1).
- Select the TestFlight-validated build (P7.2) and submit for App Store review.
- Achieve approval and release (manual or scheduled) to the public.

## 3. Non-Goals

- Building app features or new content (feature-complete before this story).
- Post-launch marketing campaigns, ASO iteration beyond initial keywords.
- Localization into multiple languages (English-first for v1.x; i18n groundwork noted).
- In-app purchases / monetization (not in scope; app is free).

## 4. Personas & User Stories

- **As a prospective player**, I want a clear, appealing listing so that I understand and download the game.
- **As the developer**, I want a complete, accurate submission so that review passes on the first attempt.
- **As an App Store reviewer**, I want metadata that matches app behavior and guidelines so that I can approve quickly.

## 5. Functional Requirements

- **FR-1.** The listing MUST include name, subtitle, promotional text, full description, and keywords accurately describing a relaxing animal logic puzzle.
- **FR-2.** Screenshots MUST be provided for all required iPhone display sizes (and iPad if the app is universal), showing real gameplay.
- **FR-3.** The app MUST have a primary category (Games → Puzzle) and an appropriate age rating (expected 4+).
- **FR-4.** Pricing MUST be set (Free) with availability territories selected.
- **FR-5.** The App Privacy answers MUST be completed as "Data Not Collected," consistent with the privacy manifest (P7.1).
- **FR-6.** Support URL (and marketing URL if used) MUST be valid; a privacy policy URL MUST be provided if required.
- **FR-7.** The submitted build MUST be a TestFlight-validated release build (P7.2) with final icon (P5.4) and no debug/test hooks.
- **FR-8.** Export compliance MUST be answered (no non-exempt encryption).
- **FR-9.** The app MUST be submitted for review and, on approval, released (manual or scheduled) to the public.

## 6. Non-Functional Requirements

- **Performance** — Release build is optimized; verified stable via P6.x and P7.2 beta.
- **Security** — No secrets or test backdoors in the shipped build; signing assets protected.
- **Privacy & Compliance** — Privacy label matches manifest; age rating accurate; meets App Review Guidelines (esp. minimum functionality, design, and safety).
- **Accessibility** — Ship the accessibility-audited build (P6.3); mention accessibility in the description where truthful.
- **Reliability** — Crash-free launch confirmed in beta before submission.
- **Internationalization** — English (US) primary locale; strings externalized so future locales are additive.
- **Maintainability** — Listing metadata and screenshot sources versioned in `docs/store/` for future updates.

## 7. Acceptance Criteria

- **AC-1.** *Given* App Store Connect, *when* the listing is reviewed, *then* name, subtitle, description, keywords, category, and URLs are complete and accurate.
- **AC-2.** *Given* the media section, *when* checked, *then* screenshots exist for all required device sizes and depict actual gameplay.
- **AC-3.** *Given* the app information, *when* reviewed, *then* age rating (4+), pricing (Free), and availability are set.
- **AC-4.** *Given* the privacy section, *when* checked, *then* answers are "Data Not Collected," matching the manifest (P7.1).
- **AC-5.** *Given* the selected build, *when* inspected, *then* it is the P7.2-validated build with final icon and no debug hooks.
- **AC-6.** *Given* submission, *when* Apple completes review, *then* the app is approved and can be released to the public.
- **AC-7.** *Given* approval, *when* released, *then* the app is downloadable from the App Store.

## 8. Data Model

No production types. Listing metadata, versioned for reuse:

```text
docs/store/
├── description.md         (name, subtitle, promo, full description, keywords)
├── screenshots/           (per-device-size PNGs)
├── age-rating.md          (questionnaire answers → 4+)
└── privacy-answers.md     (Data Not Collected, mirrors P7.1)
```

- Must ship after: P5.4 (icon), P7.1 (privacy), P7.2 (beta-validated build); QA phases P6.2–P6.4 complete.
- Must ship before: nothing — this is the final ship step.
- Shared infra needed: Apple Developer Program, App Store Connect, screenshot capture tooling.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Rejection for minimum functionality/design | M | H | Ensure multiple puzzles, themes, polish per v1.0; clear value in listing |
| Screenshot size/format mismatch | M | M | Use current required display sizes; validate before upload |
| Privacy label inconsistent with manifest | M | M | Derive answers from P7.1; single source of truth |
| Missing privacy policy URL | M | M | Host a simple static policy page; link in listing |
| Metadata guideline violation (keywords, claims) | L | M | Avoid competitor names/inflated claims; follow guidelines |

## 15. Rollout Plan

- Submit release candidate; choose manual release to control launch timing.
- On approval: release to public; monitor reviews and crash reports.
- Rollback: if a critical bug surfaces, remove from sale or expedite a fixed build; use phased release to limit exposure.
- GA criteria: approved build, listing complete, crash-free launch verified in beta.

## 16. Test Plan

- **Listing review** — Verify all metadata fields and URLs (AC-1); screenshots per size (AC-2); ratings/pricing (AC-3).
- **Privacy** — Confirm answers match manifest (AC-4).
- **Build** — Confirm selected build is P7.2 RC with icon and no debug hooks (AC-5).
- **Submission** — Submit; track review; confirm approval and release (AC-6, AC-7).
- **Manual exploratory** — Final download-from-store smoke test after release.

## 17. Documentation & Training

- Version listing copy and screenshots in `docs/store/` for future updates.
- Document the submit-and-release runbook and a post-launch monitoring checklist (reviews, crashes).

## 18. Open Questions

1. Manual vs. phased vs. automatic release on approval — recommend manual for the first public release.
2. Is a privacy policy URL strictly required for a no-data 4+ app? Recommend providing one regardless to avoid delays.
3. iPhone-only vs. universal listing — depends on P6.4 outcome; recommend iPhone-only for first release unless iPad UX is ready.
