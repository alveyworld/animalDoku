## 1. Problem Statement

Three themes exist in the catalog (P4.1) but the player cannot choose between them. A theme switcher lets players pick Frogs, Dogs, or Foxes and see the change reflected on the board immediately. Critically, switching must be **cosmetic only** — it may never alter cell states, region layout, timer, or move history, even mid-game.

## 2. Goals

- Provide a `ThemePicker` control listing the three MVP themes with icon and name.
- Reflect the selected theme on the board (animal icon, primary/accent colors) instantly.
- Persist the choice via `SettingsStore` (P4.3) so it survives relaunch.
- Guarantee switching is purely visual — no gameplay state changes.

## 3. Non-Goals

- The full settings sheet layout (P4.4) — this story delivers the picker component only.
- Theme definitions or assets (P4.1).
- Sound-toggle or high-contrast controls (P4.4, P5.3).

## 4. Personas & User Stories

- **As a player**, I want to switch animal themes so that the game matches my taste.
- **As a player mid-puzzle**, I want to change theme without losing my progress.
- **As a developer**, I want the picker bound to `SettingsStore` so selection persists automatically.

## 5. Functional Requirements

- **FR-1.** The picker MUST list exactly the three themes from `ThemeCatalog.all` with icon and name.
- **FR-2.** Selecting a theme MUST update the persisted `selectedThemeId` via `SettingsStore`.
- **FR-3.** The board and cells MUST re-render with the new theme's icon and colors immediately (no relaunch).
- **FR-4.** Switching a theme MUST NOT modify `GameSession` state (cells, history, hints, timer, completion).
- **FR-5.** The currently selected theme MUST be visually indicated in the picker.

## 6. Non-Functional Requirements

- **Performance** — Theme change re-render completes within one frame at 60fps; no puzzle re-computation.
- **Accessibility** — Each option MUST expose a VoiceOver label (theme name) and be a ≥44pt touch target; selection state announced.
- **Privacy** — Only stores a theme ID string locally.
- **Maintainability** — Picker reads from `ThemeCatalog`; adding a Future theme requires no picker changes.
- **Internationalization** — Option labels use localized theme names.

## 7. Acceptance Criteria

- **AC-1.** *Given* the picker, *when* displayed, *then* three options (Frogs, Dogs, Foxes) with icons appear.
- **AC-2.** *Given* a partially solved board, *when* the player switches theme, *then* animal icons and colors change but every cell state and the move history are unchanged.
- **AC-3.** *Given* a theme selection, *when* the app is relaunched, *then* the previously selected theme is still active.
- **AC-4.** *Given* VoiceOver, *when* focusing an option, *then* its name and selected state are announced.

## 8. Data Model

Reads/writes `selectedThemeId: String` via `SettingsStore` (P4.3). No new model.

```swift
// UI/Components/ThemePicker.swift
struct ThemePicker: View {
    @Binding var selectedThemeId: String  // bound to SettingsStore
}
```

## 9. API Surface

N/A — internal SwiftUI component; persistence via `SettingsStore`.

## 10. UI / UX

- Horizontal segmented row or grid of theme cards, each showing the animal icon and name.
- Selected card highlighted with accent border.
- Lives inside the Settings sheet (P4.4) but is a standalone reusable component.
- States: default, selected, focused (VoiceOver). No loading/error states.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Theme change mutates game state | L | H | Route only through SettingsStore; assert session unchanged in tests |
| Re-render jank on switch | L | M | Use lightweight observable theme lookup, no puzzle recompute |
| Selection not persisted | M | M | Bind directly to `@AppStorage`/SettingsStore |

## 15. Rollout Plan

- No feature flag. Ships as part of Settings sheet in v1.0.
- Rollback: hide picker; app keeps default `frogs` theme.

## 16. Test Plan

- **Unit** — Selecting a theme updates `SettingsStore.selectedThemeId`.
- **Integration** — Switch theme mid-session; assert `GameSession` cells/history identical before and after.
- **UI** — Tap each theme; verify board icon changes.
- **Accessibility** — VoiceOver announces option name and selected state.

## 17. Documentation & Training
