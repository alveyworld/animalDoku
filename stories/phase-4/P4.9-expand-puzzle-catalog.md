## 1. Problem Statement

Home (P4.8) can list and filter by Easy / Medium / Hard / Expert, but the catalog is effectively empty for most labels: only `puzzle-001` (easy) and the test fixture `puzzle-valid-4x4` ship today. Players need a small, real set of puzzles across all four difficulty labels so the filter and resume flow feel complete for v1.1.

## 2. Goals

- Author **2 new 8×8 puzzles for each difficulty** — Easy, Medium, Hard, Expert (**8 puzzles** total).
- Ship them as bundled JSON under `Resources/Puzzles/`, loadable via `PuzzleLoader` and visible on Home.
- Keep each region layout interesting: **region cell counts SHOULD vary** within a puzzle (equal-sized regions are allowed but not preferred).
- Prove each puzzle is structurally valid, rule-satisfying, and **uniquely solvable** via existing integrity tooling (P6.1).

## 3. Non-Goals

- Runtime / procedural puzzle generation (Future).
- Changing board size by difficulty (still 8×8; difficulty remains a label per GDD).
- Replacing or rewriting `puzzle-001` (it stays; these are additive).
- Difficulty mathematics beyond author judgment (no required hardness metric engine).
- UI changes beyond catalog growth (Home already lists whatever `PuzzleLoader` returns).

## 4. Personas & User Stories

- **As a player**, I want several puzzles at each difficulty so I can pick an appropriate challenge and play more than one board.
- **As a returning player**, I want Medium / Hard / Expert options so the Home filter is meaningful.
- **As a puzzle author**, I want clear verification gates so every new file is unique and ships safely.

## 5. Functional Requirements

- **FR-1.** The catalog MUST add **exactly two** new playable puzzles for each of: `easy`, `medium`, `hard`, `expert` (8 files).
- **FR-2.** Each new puzzle MUST be `size == 8`, `initialPlacements == []`, and use contiguous region ids `0…7` that partition all 64 cells exactly once.
- **FR-3.** Each region's `color` MUST be unique within its puzzle; regions SHOULD have **unequal** cell counts (at least two different sizes among the eight regions) unless equal sizes are intentional for a beginner layout.
- **FR-4.** Each puzzle's `difficulty` field MUST match its intended tier and its `id` MUST be unique across the bundle.
- **FR-5.** Each puzzle's `solution` MUST satisfy Rules 1–4 and MUST be the **only** valid solution (verified with the existing offline / integrity solver used by P6.1).
- **FR-6.** New files MUST be registered as app bundle resources (Xcode `pbxproj`) and appear in `PuzzleLoader.loadAvailablePuzzles()` / Home without loader code changes.
- **FR-7.** Recommended ids (or equivalent unique scheme):  
  `puzzle-002`, `puzzle-003` (easy); `puzzle-004`, `puzzle-005` (medium); `puzzle-006`, `puzzle-007` (hard); `puzzle-008`, `puzzle-009` (expert).
- **FR-8.** `puzzle-valid-4x4` remains a **test / UI-test fixture** and SHOULD NOT be treated as player catalog content for difficulty balance (optional: keep listed, or document that Home may show it until fixtures are excluded).

## 6. Non-Functional Requirements

- **Performance** — Load times stay negligible; uniqueness checks run at author/CI time only.
- **Reliability** — A malformed or multi-solution puzzle MUST fail integrity tests before merge.
- **Maintainability** — Pretty-printed JSON; document any authoring notes in `Resources/Puzzles/README.md`.
- **Accessibility** — Prefer distinct, accessible region colors (patterns already keyed by region id).
- **Internationalization** — Difficulty labels already localized on Home; no new UI strings required for ids.

## 7. Acceptance Criteria

- **AC-1.** *Given* the app bundle, *when* Home loads, *then* at least two puzzles appear for each of Easy, Medium, Hard, and Expert (not counting fixtures the team explicitly excludes).
- **AC-2.** *Given* each new puzzle JSON, *when* loaded via `PuzzleLoader`, *then* structural validation succeeds (partition + ids + `solution.count == 8`).
- **AC-3.** *Given* each new puzzle, *when* integrity tests run (P6.1), *then* the declared solution is valid, complete, and uniquely solvable.
- **AC-4.** *Given* each new puzzle's regions, *when* cell counts are inspected, *then* at least two regions differ in size (unless an intentional equal-region easy exception is documented in README).
- **AC-5.** *Given* filtering on Home, *when* each difficulty is selected, *then* the list shows exactly the puzzles labeled with that difficulty (including prior `puzzle-001` under Easy).

## 8. Data Model

No new types. Add eight `Puzzle` JSON artifacts conforming to the Formal Rules schema:

```json
{
  "id": "puzzle-002",
  "size": 8,
  "difficulty": "easy",
  "initialPlacements": [],
  "regions": [ /* 8 regions; variable cell counts preferred */ ],
  "solution": [ /* 8 positions, one per row */ ]
}
```

## 9. API Surface

- No new APIs. Uses `PuzzleLoader`, `Validator`, and existing `PuzzleIntegrityTests` / solver helpers.

## 10. UI / UX

- Optional polish (out of scope unless cheap): friendlier display titles derived from id.

## 11. AI / ML Considerations

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Multi-solution puzzles slip through | M | H | Gate every file with P6.1 uniqueness assertions |
| “Hard” label feels arbitrary | M | M | Author guidelines: harder = denser / more jagged regions / fewer forced early placements; peer playtest once |
| Equal-sized regions feel bland | M | L | FR-3 / AC-4 prefer variable region sizes |
| Home lists test 4×4 as Easy | L | L | Document fixture policy; optionally filter non-playable fixtures later |
| Bundle registration missed for one file | M | M | Checklist in PR; integrity suite fails if a listed file is absent |

## 15. Rollout Plan

- No feature flag; content-only. Merge when all AC pass.
- Rollback: remove individual JSON + pbxproj entries; Home still works with remaining puzzles.

## 16. Test Plan

- **Integrity (automated)** — Extend / rely on P6.1: partition, contiguous ids, unique colors, valid+complete solution, unique solution for every new id.
- **Unit** — `loadAvailablePuzzles()` returns all eight new ids; optional assertion of ≥2 puzzles per `Difficulty`.
- **Manual** — Play one Easy and one Expert end-to-end from Home; confirm filter counts.
- **Regression** — Existing `puzzle-001` and UI-test `-uiTestPuzzle puzzle-valid-4x4` path still work.

## 17. Documentation & Training

- Update `Resources/Puzzles/README.md` with the catalog table (id → difficulty → notes).
- Note authoring preference: **variable region sizes** are encouraged; equal sizes are not required by the rules.
