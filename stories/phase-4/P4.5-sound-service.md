## 1. Problem Statement

The GDD calls for calm sound effects to reinforce the relaxing feel, plus a sound toggle for accessibility. Today there is no audio layer, and actions like placing an animal or completing a puzzle are silent. A `SoundService` plays subtle SFX on key actions and strictly respects the player's sound toggle and the device silent switch.

## 2. Goals

- Provide a `SoundService` that plays calm SFX for place, remove, and win events.
- Respect `SettingsStore.soundEnabled` — no audio when disabled.
- Respect the device silent/mute state and iOS audio session conventions.
- Keep sounds short, gentle, and non-intrusive per the GDD's calm aesthetic.

## 3. Non-Goals

- Background music.
- Per-theme unique sound sets (Future) — one calm SFX set for MVP.
- Haptics (P5.6).
- Sound asset authoring beyond bundling short clips.

## 4. Personas & User Stories

- **As a player**, I want gentle sound feedback so that actions feel satisfying.
- **As a player in a quiet space**, I want to disable sound so that the game stays silent.
- **As a developer**, I want a single service so that any view can trigger a sound by name.

## 5. Functional Requirements

- **FR-1.** The service MUST play distinct calm SFX for place, remove, and win events.
- **FR-2.** The service MUST NOT play any sound when `SettingsStore.soundEnabled` is `false`.
- **FR-3.** The service MUST honor the device silent switch and not interrupt other apps' audio (ambient/playback category, non-mixing-hostile).
- **FR-4.** Playback MUST be non-blocking and MUST NOT delay UI feedback.
- **FR-5.** Missing or failed audio assets MUST fail silently without crashing.

## 6. Non-Functional Requirements

- **Performance** — Sounds preloaded/cached; latency between action and audio < 50ms.
- **Security** — No network audio.
- **Privacy** — No microphone use, no data collection.
- **Accessibility** — Sound is supplementary; all feedback also has a visual counterpart. Toggle satisfies GDD sound-toggle requirement.
- **Reliability** — Absent assets degrade gracefully.
- **Maintainability** — Sounds referenced by an enum, not string literals scattered in views.

## 7. Acceptance Criteria

- **AC-1.** *Given* sound enabled, *when* an animal is placed, *then* the place SFX plays once.
- **AC-2.** *Given* sound disabled, *when* any action occurs, *then* no sound plays.
- **AC-3.** *Given* the puzzle is completed, *when* the win state triggers, *then* the win SFX plays once.
- **AC-4.** *Given* a missing audio file, *when* triggered, *then* no crash occurs and the action still completes.
- **AC-5.** *Given* the device is on silent, *when* a sound triggers, *then* behavior matches the chosen audio session category.

## 8. Data Model

```swift
// Services/SoundService.swift
enum GameSound { case place, remove, win }

protocol SoundPlaying { func play(_ sound: GameSound) }

final class SoundService: SoundPlaying {
    init(settings: SettingsStore)
    func play(_ sound: GameSound)  // no-op when soundEnabled == false
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Audio session interrupts music apps | M | M | Use ambient category; do not deactivate others |
| SFX feels jarring, breaks calm feel | M | M | Design short, soft clips; volume-limit |
| Latency between action and sound | L | M | Preload/cache players at init |

## 15. Rollout Plan

- No feature flag. Default sound on; toggle in settings.
- Rollback: disable service call sites; game stays silent.

## 16. Test Plan

- **Unit** — `play` is a no-op when `soundEnabled` is false; correct sound requested per event.
- **Integration** — Place/remove/win trigger the expected `GameSound`.
- **Manual** — Verify calm feel, silent-switch behavior, and no music interruption on device.
