## 1. Problem Statement

Today, quitting the app mid-puzzle loses all progress. The `SaveGame` schema is already defined in the Formal Rules, and the GDD schedules in-progress persistence for v1.1. A `SaveGameStore` persists the active attempt (cells, timer, hints, completion) so players can resume exactly where they left off.

## 2. Goals

- Persist the in-progress attempt as a `SaveGame` per the defined schema.
- Restore the exact board, elapsed time, and hint count on relaunch.
- Save automatically at safe points (backgrounding, meaningful state change).
- Clear or overwrite the save on completion or reset appropriately.

## 3. Non-Goals

- Multiple concurrent saves across many puzzles (single active save for MVP of this feature; keyed by `puzzleId`).
- Cloud sync (Future).
- Preferences persistence (that is P4.3).
- Undo/redo stack persistence — only board state and counters are restored.

## 4. Personas & User Stories

- **As a player**, I want my puzzle progress saved so that I can resume after closing the app.
- **As a player**, I want a completed puzzle to reflect its completed state on return.
- **As a developer**, I want a `SaveGame` round-trip so persistence is fully testable.

## 5. Functional Requirements

- **FR-1.** The store MUST serialize the current attempt to a `SaveGame` (Codable) keyed by `puzzleId`.
- **FR-2.** The store MUST save automatically when the app backgrounds and on significant state changes.
- **FR-3.** On launch, the store MUST restore the saved attempt if one exists for the puzzle being opened.
- **FR-4.** Restoring MUST reproduce the exact `cells`, `elapsedSeconds`, `hintsUsed`, and `completed` values.
- **FR-5.** Reset MUST clear or reinitialize the save for that puzzle.
- **FR-6.** A corrupt or version-mismatched save MUST be discarded gracefully (start fresh), never crash.

## 6. Non-Functional Requirements

- **Performance** — Save/restore of a single 8×8 attempt is < 10ms; saves are debounced/off the main path if needed.
- **Security** — Local file/`UserDefaults`; no sensitive data.
- **Privacy** — On-device only; no collection.
- **Reliability** — Atomic writes; partial writes never corrupt a valid prior save.
- **Backward compatibility** — Include a schema version; unknown versions are discarded safely.
- **Maintainability** — Single `Services/SaveGameStore.swift`; JSON via `Codable`.

## 7. Acceptance Criteria

- **AC-1.** *Given* a partially solved puzzle, *when* the app is killed and relaunched, *then* the exact board, timer, and hints are restored.
- **AC-2.** *Given* a completed puzzle, *when* relaunched, *then* the board shows completed/locked state.
- **AC-3.** *Given* reset, *when* invoked, *then* the persisted save for that puzzle is cleared.
- **AC-4.** *Given* a corrupt save file, *when* loaded, *then* the app starts a fresh attempt without crashing.
- **AC-5.** *Given* a `SaveGame` value, *when* encoded then decoded, *then* it round-trips identically.

## 8. Data Model

Uses `SaveGame` from [Formal Rules](../../AnimalDoku_Formal_Rules_and_Data_Model.md#savegame-v11):

```swift
struct SaveGame: Codable {
    let puzzleId: String
    let elapsedSeconds: Int
    let cells: [Cell]
    let hintsUsed: Int
    let mistakes: Int      // 0 for MVP (mistake count deferred)
    let completed: Bool
}

// Services/SaveGameStore.swift
protocol SaveGamePersisting {
    func save(_ game: SaveGame)
    func load(puzzleId: String) -> SaveGame?
    func clear(puzzleId: String)
}
```

Add a stored `schemaVersion` alongside the payload for safe migration.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Corrupt save crashes launch | M | H | Try/catch decode; discard and start fresh |
| Restored state diverges from live rules | L | H | Re-validate restored board on load |
| Save races on rapid background/foreground | M | M | Debounce and use atomic write |
| Schema drift in future | M | M | Persist `schemaVersion`; discard unknown |

## 15. Rollout Plan

- Ships in v1.1. Feature flag `saveGameEnabled` (default off until validated), then on.
- Rollback: disable flag; app behaves as MVP (no resume).

## 16. Test Plan

- **Unit** — `SaveGame` Codable round-trip; store save/load/clear; corrupt payload discarded.
- **Integration** — Save on background, restore on relaunch reproduces exact state; reset clears save.
- **Manual** — Kill app mid-puzzle across states (partial, completed) and confirm resume.

## 17. Documentation & Training

- Document save location, schema version policy, and resume behavior.
