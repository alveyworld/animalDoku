## 1. Problem Statement

Players need one place to change their theme, toggle sound, and enable high contrast. The GDD lists a modal Settings sheet as an MVP screen reachable from the Game screen. Without it, the theme switcher and accessibility toggles have no home in the UI.

## 2. Goals

- Present a modal Settings sheet reachable from the Game screen.
- Include the theme picker (P4.2), a sound toggle, and a high-contrast toggle.
- Bind all controls to `SettingsStore` (P4.3) so changes persist immediately.
- Match the GDD's soft, minimal visual style.

## 3. Non-Goals

- The theme picker component itself (P4.2) — this story composes it.
- Persistence mechanics (P4.3) or sound playback (P4.5) or contrast rendering (P5.3).
- Puzzle selection or difficulty controls (P4.8).

## 4. Personas & User Stories

- **As a player**, I want a settings screen so that I can adjust theme, sound, and contrast in one place.
- **As a player using accessibility settings**, I want a clear high-contrast toggle.
- **As a developer**, I want a sheet that only reads/writes `SettingsStore` so it holds no state of its own.

## 5. Functional Requirements

- **FR-1.** The Game screen MUST provide a control to present the Settings sheet modally.
- **FR-2.** The sheet MUST embed the `ThemePicker` (P4.2).
- **FR-3.** The sheet MUST include a sound on/off toggle bound to `SettingsStore.soundEnabled`.
- **FR-4.** The sheet MUST include a high-contrast toggle bound to `SettingsStore.highContrastEnabled`.
- **FR-5.** Changes MUST take effect immediately and persist (via P4.3) without an explicit save action.
- **FR-6.** The sheet MUST be dismissible and return the player to the Game screen with game state unchanged.

## 6. Non-Functional Requirements

- **Performance** — Sheet presents/dismisses smoothly; toggles apply within one frame.
- **Accessibility** — All controls ≥44pt, VoiceOver-labeled, grouped logically; Dynamic Type supported.
- **Privacy** — No data collection.
- **Maintainability** — Sheet holds no independent state; all values flow through `SettingsStore`.
- **Internationalization** — All labels localized.

## 7. Acceptance Criteria

- **AC-1.** *Given* the Game screen, *when* the settings control is tapped, *then* the Settings sheet appears modally.
- **AC-2.** *Given* the sheet, *when* the sound toggle is flipped, *then* `SettingsStore.soundEnabled` updates and persists.
- **AC-3.** *Given* the sheet, *when* a theme is selected, *then* the board behind reflects it after dismissal (or live).
- **AC-4.** *Given* the sheet is dismissed, *when* returning to the game, *then* the puzzle state is unchanged.
- **AC-5.** *Given* VoiceOver, *when* traversing the sheet, *then* every control is labeled and reachable.

## 8. Data Model

No new model. Binds to `SettingsStore` (P4.3):

```swift
// UI/Screens/SettingsView.swift
struct SettingsView: View {
    @Environment(SettingsStore.self) var settings
}
```

## 10. UI / UX

- Modal sheet with sections: **Theme** (picker), **Sound** (toggle), **Accessibility** (high-contrast toggle).
- Standard navigation title "Settings" with a Done/close affordance.
- States: default only; no loading/error (all local). Portrait layout.
- Copy keys: `settings.title`, `settings.sound`, `settings.highContrast`, `settings.theme`.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Toggle changes lost on dismiss | L | M | Bind directly to store; write immediately |
| Sheet holds duplicate state | M | M | No local `@State` for values; use store bindings |
| Cramped layout at large Dynamic Type | M | M | Use `Form`/scrollable list; test largest sizes |

## 15. Rollout Plan
