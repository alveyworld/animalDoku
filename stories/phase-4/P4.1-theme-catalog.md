## 1. Problem Statement

The GDD locks three cosmetic themes for v1.0 — Frogs, Dogs, Foxes — but there is no central definition of theme metadata (icon, colors, name). Without a `ThemeCatalog`, views hard-code a single animal and cannot honor the player's theme choice. Themes are the primary personalization surface and must exist before the switcher (P4.2) or settings sheet (P4.4) can consume them.

## 2. Goals

- Define a static `ThemeCatalog` exposing the three locked MVP themes keyed by ID (`frogs`, `dogs`, `foxes`).
- Map each theme to its animal icon asset, display name, primary color, and accent color per the `Theme` model.
- Provide a single lookup so any view or view model can resolve a theme by ID with a guaranteed default fallback (`frogs`).
- Keep themes strictly cosmetic — no theme value may influence puzzle logic, cell positions, or validation.

## 3. Non-Goals

- Theme switcher UI (P4.2) or settings persistence (P4.3).
- Future themes (Birds, Rabbits, Bears) — out of MVP scope.
- Per-theme sound sets beyond the calm SFX defined in P4.5.
- Downloadable or unlockable themes.

## 4. Personas & User Stories

- **As a player**, I want to choose an animal theme so that the game feels personal without changing difficulty.
- **As a developer**, I want a single source of theme truth so that views resolve icons and colors consistently.
- **As a designer**, I want three complete, distinct themes so that the catalog matches the locked GDD table.

## 5. Functional Requirements

- **FR-1.** The catalog MUST define exactly three themes with IDs `frogs`, `dogs`, `foxes` matching the GDD table.
- **FR-2.** Each `Theme` MUST provide `id`, `name`, `animal`, `icon` (asset name), `primaryColor`, and `accentColor`.
- **FR-3.** The catalog MUST expose a lookup `theme(id:)` that returns the matching theme or the default (`frogs`) when the ID is unknown.
- **FR-4.** The default theme MUST be `frogs` per GDD v0.1.
- **FR-5.** Themes MUST NOT expose any property that alters puzzle state, region layout, or validation.
- **FR-6.** Icons MUST resolve to asset-catalog names created in P1.5; code SHOULD fall back to an SF Symbol if an asset is missing so the app never renders a blank cell.

## 6. Non-Functional Requirements

- **Performance** — Catalog is a compile-time constant; lookups are O(1) with no I/O.
- **Security** — No network, no user data.
- **Privacy** — No data collection.
- **Accessibility** — Each theme's primary/accent pair SHOULD support the ≥4.5:1 contrast target used by P5.3; animal `name` is available for VoiceOver labels.
- **Maintainability** — All themes defined in one file (`Themes/ThemeCatalog.swift`); colors reference design tokens where practical.
- **Internationalization** — Theme display `name` MUST be a localizable string key.

## 7. Acceptance Criteria

- **AC-1.** *Given* `ThemeCatalog.all`, *when* read, *then* exactly three themes with IDs `frogs`, `dogs`, `foxes` are returned in a stable order.
- **AC-2.** *Given* `ThemeCatalog.theme(id: "dogs")`, *when* called, *then* the Dogs theme is returned with a non-empty icon and distinct colors.
- **AC-3.** *Given* an unknown ID, *when* `theme(id:)` is called, *then* the `frogs` default is returned.
- **AC-4.** *Given* any theme, *when* applied, *then* the puzzle's cells, regions, and solution are unchanged.

## 8. Data Model

Uses the `Theme` struct from [Formal Rules](../../AnimalDoku_Formal_Rules_and_Data_Model.md#theme):

```swift
struct Theme: Identifiable {
    let id: String
    let name: String          // localized display name
    let animal: String
    let icon: String          // asset name, e.g. "theme-frogs-icon"
    let primaryColor: String  // hex
    let accentColor: String   // hex
}

enum ThemeCatalog {
    static let all: [Theme]        // [frogs, dogs, foxes]
    static let defaultTheme: Theme // frogs
    static func theme(id: String) -> Theme
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Missing icon assets crash or blank cells | M | M | SF Symbol fallback in rendering code |
| Theme colors fail contrast target | M | M | Validate pairs against P5.3 ≥4.5:1 before merge |
| Theme accidentally leaks into logic | L | H | Keep `Theme` free of any board/solution fields; review in PR |

## 15. Rollout Plan

- No feature flag. Merge with `frogs` as default; switcher (P4.2) exposes the rest.
- Rollback: revert catalog; UI falls back to default theme constant.
