## 1. Problem Statement

Players need quick access to undo, redo, reset, and hint actions during a puzzle. These engine capabilities exist in P3.3–P3.6 but have no UI affordances. Without `GameToolbar`, the game is unplayable beyond raw cell taps — players cannot recover from mistakes or request assistance.

## 2. Goals

- Render four toolbar buttons: Undo, Redo, Reset, Hint.
- Disable each button when its action is unavailable (`canUndo`, `canRedo`, `canReset`, `canHint`).
- Wire button taps to `GameViewModel` intents (P3.13).
- Meet accessibility requirements: VoiceOver labels, ≥ 44×44 pt targets.

## 3. Non-Goals

- Input mode toggle (P3.11 — separate component).
- Settings access button (P4.4).
- Timer display (tracked in session; displayed v1.0, P4.6).
- Custom toolbar styling per theme (v0.1 uses design tokens; theme accents in P4.1).

## 4. Personas & User Stories

- **As a player**, I want an Undo button so I can revert my last move.
- **As a player**, I want a Redo button so I can reapply an undone move.
- **As a player**, I want a Reset button so I can start the puzzle over.
- **As a player**, I want a Hint button so I can reveal one correct cell when stuck.
- **As a player using VoiceOver**, I want each button labeled and its enabled/disabled state announced.

## 5. Functional Requirements

- **FR-1.** `GameToolbar` MUST render four buttons: Undo, Redo, Reset, Hint.
- **FR-2.** Undo button MUST be disabled when `canUndo == false` (empty undo stack).
- **FR-3.** Redo button MUST be disabled when `canRedo == false` (empty redo stack).
- **FR-4.** Reset button MUST be enabled whenever the board is not locked by completion; MUST be disabled when puzzle is complete (P3.15).
- **FR-5.** Hint button MUST be disabled when `canHint == false` (`hintsUsed >= 3`, no valid target, or board locked).
- **FR-6.** Tapping Undo MUST invoke `viewModel.undo()`.
- **FR-7.** Tapping Redo MUST invoke `viewModel.redo()`.
- **FR-8.** Tapping Reset MUST invoke `viewModel.reset()` (with optional confirmation alert — see Open Questions).
- **FR-9.** Tapping Hint MUST invoke `viewModel.requestHint()`.
- **FR-10.** Each button MUST have a VoiceOver accessibility label and hint describing its action.
- **FR-11.** Touch targets MUST be ≥ 44×44 pt (P1.4 `TouchTarget.minimum`).
- **FR-12.** Hint button SHOULD display remaining hints (e.g. "Hint (2)") when `hintsRemaining > 0`.

## 6. Non-Functional Requirements

- **Performance** — Instant tap response; no debounce needed.
- **Accessibility** — VoiceOver labels (FR-10); disabled state announced; SF Symbols with accessibility labels.
- **Maintainability** — `UI/Components/GameToolbar.swift`; pure presentation, no engine imports.
- **Internationalization** — Button labels via `LocalizedStringKey` placeholders for v1.1.

## 7. Acceptance Criteria

- **AC-1.** *Given* an empty undo stack, *when* the toolbar renders, *then* Undo is disabled.
- **AC-2.** *Given* at least one action on the undo stack, *when* Undo is tapped, *then* the last action is reverted (P3.3).
- **AC-3.** *Given* actions on the redo stack after undo, *when* Redo is tapped, *then* the action is reapplied (P3.4).
- **AC-4.** *Given* a board with moves, *when* Reset is tapped, *then* the board returns to initial empty state (P3.5).
- **AC-5.** *Given* `hintsUsed < 3` and a valid target, *when* Hint is tapped, *then* one cell is revealed (P3.6).
- **AC-6.** *Given* `hintsUsed == 3`, *when* the toolbar renders, *then* Hint is disabled.
- **AC-7.** *Given* a completed puzzle, *when* the toolbar renders, *then* Undo, Redo, Reset, and Hint are all disabled.
- **AC-8.** *Given* VoiceOver on the Undo button, *when* focused, *then* "Undo" is announced with enabled/disabled state.

## 8. Data Model

Presentation-only; state from `GameViewModel` (P3.13):

```swift
// UI/Components/GameToolbar.swift
struct GameToolbar: View {
    let canUndo: Bool
    let canRedo: Bool
    let canReset: Bool
    let canHint: Bool
    var hintsRemaining: Int = 3
  var isEnabled: Bool = true          // false when board locked

    var onUndo: () -> Void = {}
    var onRedo: () -> Void = {}
    var onReset: () -> Void = {}
    var onHint: () -> Void = {}

    var body: some View { /* HStack of IconButtons */ }
}
```

SF Symbol mapping (recommended):

| Button | Symbol | Accessibility Label |
|--------|--------|---------------------|
| Undo | `arrow.uturn.backward` | "Undo" |
| Redo | `arrow.uturn.forward` | "Redo" |
| Reset | `arrow.counterclockwise` | "Reset puzzle" |
| Hint | `lightbulb` | "Hint, N remaining" |

## 9. API Surface

N/A — SwiftUI component. Callback-based API (no direct `GameSession` access).

## 10. UI / UX

**Layout:** Horizontal row below the board (or above, per GameView layout in P3.14). Equal spacing between buttons; centered in available width.

**Visual states:**

| Button | Enabled | Disabled |
|--------|---------|----------|
| Undo | Accent color icon | Gray icon, no tap |
| Redo | Accent color icon | Gray icon, no tap |
| Reset | Accent color icon | Gray icon, no tap |
| Hint | Accent + count badge | Gray icon, no tap |

**Reset confirmation (optional):** Alert "Reset puzzle?" with Cancel / Reset actions — recommended if accidental resets are common; can ship without alert for v0.1 speed.

**Previews:** `#Preview` showing enabled and disabled states.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Accidental reset clears progress | M | M | Optional confirmation alert; Play Again on win screen (P3.15) |
| Hint button enabled when no valid target | M | M | `canHint` computed in ViewModel includes target check (P3.6) |
| Toolbar cramped on SE | M | M | Icon-only buttons; test on 375 pt width |
| Disabled buttons still tappable | L | H | SwiftUI `.disabled()` modifier on each button |

## 15. Rollout Plan

- No feature flag. Always visible on Game screen.
- Merge when AC pass with P3.14 integration.
- Rollback: remove toolbar; engine actions still testable via unit tests.

## 16. Test Plan

- **Unit** — N/A (presentation); `canUndo`/`canRedo`/`canHint` logic tested in P3.7.
- **UI test** — Tap Undo after place → cell empty; tap Hint → cell revealed; tap Reset → board cleared (T-082).
- **Manual** — All four buttons in enabled/disabled states; VoiceOver walkthrough (AC-8).
- **Accessibility** — Labels, disabled state announcement, 44 pt targets.
