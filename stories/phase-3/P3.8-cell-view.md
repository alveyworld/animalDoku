## 1. Problem Statement

The board needs a single reusable cell component that renders all visual states — empty, blocked (X), animal placed, selected, and rule violation — with accessible touch targets. Without `CellView`, `BoardView` cannot display the puzzle grid.

## 2. Goals

- Render the three `CellState` values distinctly: `.empty`, `.blocked`, `.animal`.
- Support overlay states: selected, violation highlight.
- Meet ≥ 44×44 pt touch target (GDD accessibility).
- Provide VoiceOver labels announcing row, column, region, and state.
- Use design tokens from P1.4 (colors, spacing, corner radius).

## 3. Non-Goals

- Grid layout / region backgrounds (P3.9).
- Colorblind region patterns (P3.10).
- Tap handling logic (routed by `BoardView`/`GameViewModel` in P3.14).
- Theme animal icon assets (v0.1 uses default Frogs theme placeholder or SF Symbol until P4.1).

## 4. Personas & User Stories

- **As a player**, I want to see clearly whether a cell is empty, marked X, or has an animal.
- **As a player**, I want violating cells highlighted so I understand rule conflicts.
- **As a player using VoiceOver**, I want each cell announced with position and state.
- **As a developer**, I want a parameterized `CellView` usable in previews and `BoardView`.

## 5. Functional Requirements

- **FR-1.** `CellView` MUST render `.empty` as a clear/neutral cell (region background shows through from parent).
- **FR-2.** `CellView` MUST render `.blocked` with a visible X mark (not color alone).
- **FR-3.** `CellView` MUST render `.animal` with the current theme's animal icon (placeholder acceptable for v0.1).
- **FR-4.** When `isSelected == true`, the cell MUST show a selection border or ring.
- **FR-5.** When `isViolating == true`, the cell MUST show a violation highlight using `AppColors.error` / violation token from P1.4.
- **FR-6.** The tappable area MUST be ≥ 44×44 pt (`TouchTarget.minimum` from P1.4).
- **FR-7.** VoiceOver MUST announce: "Row {r}, Column {c}, Region {id}, {state}" (e.g. "empty", "blocked", "animal", "violation").
- **FR-8.** `CellView` MUST accept parameters without importing `GameSession` (presentation-only).

## 6. Non-Functional Requirements

- **Performance** — 64 instances on screen at 60fps; avoid heavy shadows/blur.
- **Accessibility** — WCAG-minded contrast for X and animal icon on region backgrounds; VoiceOver (FR-7).
- **Maintainability** — SwiftUI struct in `UI/Components/`; preview variants for all states.
- **Internationalization** — VoiceOver strings use format placeholders; no hard-coded English in production without `LocalizedStringKey` (full i18n v1.1).

## 7. Acceptance Criteria

- **AC-1.** *Given* `state == .empty`, *when* previewed, *then* no X or animal icon is shown.
- **AC-2.** *Given* `state == .blocked`, *when* previewed, *then* an X mark is visible and distinguishable from empty without relying on color alone.
- **AC-3.** *Given* `state == .animal`, *when* previewed, *then* an animal icon is centered in the cell.
- **AC-4.** *Given* `isViolating == true`, *when* previewed, *then* a violation highlight is visible on that cell.
- **AC-5.** *Given* VoiceOver on a cell, *when* focused, *then* position and state are announced per FR-7.
- **AC-6.** *Given* the cell frame, *when* measured on iPhone SE, *then* touch target ≥ 44 pt.

## 8. Data Model

Presentation model (no persistence):

```swift
// UI/Components/CellView.swift
struct CellView: View {
    let row: Int
    let col: Int
    let regionId: Int
    let state: CellState
    var isSelected: Bool = false
    var isViolating: Bool = false
    var animalIcon: Image              // from theme; default frog placeholder
    var onTap: () -> Void = {}
}
```

Violation positions come from `ValidationResult.violations` (P2.7), computed in `GameViewModel` (P3.13).

## 10. UI / UX

**Visual states:**

| State | Appearance |
|-------|------------|
| Empty | Transparent/minimal; region color from parent |
| Blocked | X mark, medium weight stroke |
| Animal | Centered theme icon, gentle scale on place (animation in P5.1) |
| Selected | 2pt accent border |
| Violation | Red/error border or background tint |

**Previews:** SwiftUI `#Preview` grid showing all 3 states × selected × violating combinations.

**Reduce Motion:** No mandatory animation in this story; static states only.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| X mark invisible on some region colors | M | M | Use contrasting stroke; test on all 8 region colors |
| Touch target too small on SE | M | H | Parent `BoardView` enforces min size; AC-6 test |
| Violation highlight clashes with region color | M | M | Use border overlay, not fill-only |

## 15. Rollout Plan
