## 1. Problem Statement

`BoardView` (P3.9) paints regions using color alone. GDD and Formal Rules require colorblind-friendly region distinction — each of the 8 regions must be identifiable without relying on hue. Without pattern overlays, players with color-vision deficiency cannot reliably tell regions apart, blocking a core accessibility requirement for v0.1.

## 2. Goals

- Overlay a distinct visual pattern on each region so regions are distinguishable in grayscale.
- Integrate patterns into `BoardView` without breaking region background colors.
- Assign patterns deterministically by `regionId` (0–7 for MVP).
- Respect Reduce Motion (static patterns only; no animation).

## 3. Non-Goals

- High contrast mode styling (P5.3).
- User-toggle for patterns (always on in v0.1 per GDD accessibility).
- Pattern customization or per-puzzle pattern assignment (patterns are derived from `regionId`).
- Settings UI for accessibility (P4.4).

## 4. Personas & User Stories

- **As a player with color-vision deficiency**, I want regions distinguishable by pattern so I can solve puzzles without guessing region boundaries.
- **As a player**, I want patterns subtle enough not to distract from animal icons and X marks.
- **As a developer**, I want a reusable `RegionPattern` view that `BoardView` composes per cell.
- **As a QA tester**, I want grayscale simulation to confirm all 8 regions are visually distinct.

## 5. Functional Requirements

- **FR-1.** Each region (`regionId` 0…`size-1`) MUST have a unique overlay pattern in addition to its background color.
- **FR-2.** Patterns MUST remain distinguishable when the board is viewed in grayscale (simulated deuteranopia/protanopia).
- **FR-3.** `RegionPattern` MUST accept `regionId: Int` and render the corresponding pattern tile clipped to the cell bounds.
- **FR-4.** Pattern opacity MUST be low enough that region background color remains visible (recommended 15–25% opacity overlay).
- **FR-5.** Patterns MUST NOT obscure the animal icon, X mark, selection ring, or violation highlight from `CellView` (P3.8).
- **FR-6.** `BoardView` MUST compose `RegionPattern` behind `CellView` content for every cell.
- **FR-7.** Pattern assignment MUST be deterministic: `regionId % patternCount` maps to a fixed pattern set of at least 8 distinct patterns.
- **FR-8.** Patterns SHOULD use shape/border differentiation (dots, stripes, crosshatch, diagonal lines, etc.) — not color alone.

## 6. Non-Functional Requirements

- **Performance** — 64 pattern overlays at 60fps; prefer lightweight `Canvas` or tiled `Image` assets over complex shaders.
- **Accessibility** — Meets GDD colorblind-friendly regions requirement; complements VoiceOver region announcements (P3.8).
- **Maintainability** — `UI/Components/RegionPattern.swift`; pattern definitions centralized in one enum or catalog.
- **Internationalization** — N/A (visual only).
- **Scalability** — Pattern catalog extensible if future puzzles use `size > 8`.

## 7. Acceptance Criteria

- **AC-1.** *Given* an 8×8 board with 8 regions, *when* viewed in grayscale (simulator color filters), *then* all 8 regions are visually distinguishable.
- **AC-2.** *Given* a cell with `.animal` state, *when* rendered, *then* the animal icon is clearly visible over the pattern.
- **AC-3.** *Given* a cell with `.blocked` state, *when* rendered, *then* the X mark is clearly visible over the pattern.
- **AC-4.** *Given* `regionId == 3`, *when* any cell in that region is rendered, *then* it shows the same pattern as other cells in region 3.
- **AC-5.** *Given* `regionId == 0` and `regionId == 1`, *when* compared side by side, *then* their patterns are visually distinct.
- **AC-6.** *Given* Reduce Motion enabled, *when* the board is displayed, *then* patterns render statically with no animation.

## 8. Data Model

No persistence. Pattern catalog is static:

```swift
// UI/Components/RegionPattern.swift
enum RegionPatternStyle: CaseIterable {
    case dots, horizontalStripes, verticalStripes, diagonalStripes
    case crosshatch, grid, chevrons, circles
    // ≥ 8 cases for MVP

    static func forRegion(_ regionId: Int) -> RegionPatternStyle {
        allCases[regionId % allCases.count]
    }
}

struct RegionPattern: View {
    let regionId: Int
    var opacity: Double = 0.2

    var body: some View { /* render pattern tile */ }
}
```

Integration in `BoardView`:

```swift
ZStack {
    regionColorBackground
    RegionPattern(regionId: cell.regionId)
    CellView(/* ... */)
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Patterns too strong, obscure icons | M | M | FR-4 opacity cap; AC-2/AC-3 visual QA |
| Two patterns look similar in grayscale | M | H | AC-1 grayscale test; use shape-diverse set |
| Performance hit from 64 Canvas draws | L | M | Profile on SE; cache pattern as `Image` if needed |
| Pattern clashes with violation red | L | M | Violation uses border overlay on top (P3.8) |

## 15. Rollout Plan

- No feature flag. Patterns always on for v0.1 (GDD accessibility).
- Merge when AC pass and grayscale simulation confirms distinguishability.
- Rollback: remove `RegionPattern` overlay; board falls back to color-only (degraded a11y).

## 16. Test Plan

- **Unit** — `RegionPatternStyle.forRegion(_)` returns distinct styles for ids 0–7.
- **Snapshot / preview** — All 8 patterns on representative region colors; animal and X legibility.
- **Manual** — iOS Simulator Accessibility Inspector → color filters (grayscale, deuteranopia); confirm AC-1.
- **Accessibility** — Grayscale distinguishability (AC-1); no impact on VoiceOver (patterns are decorative).

## 17. Documentation & Training

- Document pattern-to-regionId mapping in `RegionPattern.swift` header.
- Note GDD accessibility requirement in code comment.

## 18. Open Questions

- Vector `Canvas` vs. raster assets: prefer `Canvas` for v0.1 (no asset pipeline); switch to assets if performance issues arise.
- Pattern stroke color: fixed dark gray vs. adaptive to region luminance — default to semi-transparent black (0.2 opacity).
