## 1. Problem Statement

`GameSession` (P3.1–P3.6) is a pure-Swift engine type with no SwiftUI awareness. SwiftUI views need an observable bridge that exposes session state, routes user intents to session methods, and triggers view updates on every board mutation. Without `GameViewModel`, `GameView` cannot bind to live game state or react to completion.

## 2. Goals

- Implement `@Observable class GameViewModel` wrapping a `GameSession`.
- Expose read-only derived state for UI binding: cells, validation, completion, undo/redo/hint availability.
- Route cell taps through `inputMode` to the correct session action (P3.2).
- Trigger win-screen presentation when `completed` becomes true (P3.15).
- Keep the ViewModel thin — delegate all logic to `GameSession` / `HintService`.

## 3. Non-Goals

- Persistence or save-game restore (v1.1, P4.7).
- Theme selection logic (P4.2; v0.1 uses default Frogs theme).
- Timer ticking service (P4.6; `elapsedSeconds` field exists but not displayed in v0.1).
- Sound effect playback (P4.5).
- Settings store integration (P4.3).

## 4. Personas & User Stories

- **As a developer**, I want a single ViewModel per game screen so SwiftUI views have one injection point.
- **As a developer**, I want UI to auto-refresh when the board changes without manual `objectWillChange` boilerplate.
- **As a player** (indirect), I want the board, toolbar, and win overlay to stay in sync with my actions instantly.
- **As a tester**, I want ViewModel methods callable from unit tests with a mock puzzle.

## 5. Functional Requirements

- **FR-1.** `GameViewModel` MUST be annotated `@Observable` (iOS 17+ Observation framework).
- **FR-2.** `GameViewModel` MUST initialize with a `Puzzle` (loads default puzzle for v0.1 via `PuzzleLoader`, P2.3).
- **FR-3.** The ViewModel MUST expose `cells`, `puzzle`, `validationResult`, `inputMode`, `selectedPosition`, `hintsUsed`, and `isCompleted` for UI binding.
- **FR-4.** The ViewModel MUST expose computed properties: `canUndo`, `canRedo`, `canHint`, `hintsRemaining`, `violatingPositions: Set<Position>`.
- **FR-5.** `handleCellTap(at:)` MUST route to place/remove (Place mode) or toggle-blocked (Mark mode) per Formal Rules (P3.2).
- **FR-6.** `handleCellTap(at:)` MUST be a no-op when `isCompleted == true` (board locked, P3.15).
- **FR-7.** `undo()`, `redo()`, `reset()`, and `requestHint()` MUST delegate to `GameSession` equivalents (P3.3–P3.6).
- **FR-8.** `reset()` MUST clear win state and dismiss win overlay readiness.
- **FR-9.** When `session.completed` transitions to `true`, the ViewModel MUST set `showWinScreen = true`.
- **FR-10.** `selectedPosition` SHOULD update on cell tap for hint targeting (P3.6 FR-2).
- **FR-11.** The ViewModel MUST NOT import UIKit; SwiftUI import is acceptable for `@Observable`.

## 6. Non-Functional Requirements

- **Performance** — View updates on mutation only; avoid redundant validation calls (session handles revalidation).
- **Maintainability** — `ViewModels/GameViewModel.swift`; no game logic duplicated from engine.
- **Testability** — Initializable with injected `GameSession` for unit tests.
- **Reliability** — Single source of truth: all mutations go through session methods.

## 7. Acceptance Criteria

- **AC-1.** *Given* a ViewModel with a fresh session, *when* observed by a SwiftUI view, *then* 64 empty cells render.
- **AC-2.** *Given* Place mode, *when* `handleCellTap` on an empty cell, *then* that cell becomes `.animal` and the view updates.
- **AC-3.** *Given* Mark mode, *when* `handleCellTap` on an empty cell, *then* that cell becomes `.blocked`.
- **AC-4.** *Given* a placed animal, *when* `undo()` is called, *then* the cell returns to prior state and `canRedo == true`.
- **AC-5.** *Given* a board matching the solution, *when* the last animal is placed, *then* `isCompleted == true` and `showWinScreen == true`.
- **AC-6.** *Given* a completed board, *when* `handleCellTap` is called, *then* no state change occurs.
- **AC-7.** *Given* `reset()` after completion, *when* inspected, *then* board is empty, `isCompleted == false`, `showWinScreen == false`.
- **AC-8.** *Given* violations on the board, *when* `violatingPositions` is read, *then* it contains all positions from `validationResult.violations`.

## 8. Data Model

```swift
// ViewModels/GameViewModel.swift
@Observable
final class GameViewModel {
    private let session: GameSession

    // Published state (via @Observable)
    var inputMode: InputMode {
        get { session.inputMode }
        set { session.inputMode = newValue }
    }
    var selectedPosition: Position?
    var showWinScreen: Bool = false

    // Read-only derived
    var puzzle: Puzzle { session.puzzle }
    var cells: [Cell] { session.cells }
    var validationResult: ValidationResult { session.validationResult }
    var isCompleted: Bool { session.completed }
    var hintsUsed: Int { session.hintsUsed }
    var canUndo: Bool { !session.undoStack.isEmpty }
    var canRedo: Bool { !session.redoStack.isEmpty }
    var canHint: Bool { /* session.canHint */ }
    var hintsRemaining: Int { HintService.maxHints - session.hintsUsed }
    var violatingPositions: Set<Position> { /* flatten violations */ }

    init(puzzle: Puzzle) {
        self.session = GameSession(puzzle: puzzle)
    }

    func handleCellTap(at position: Position) { /* route by inputMode */ }
    func undo() { session.undo(); checkWinState() }
    func redo() { session.redo(); checkWinState() }
    func reset() { session.reset(); showWinScreen = false; selectedPosition = nil }
    func requestHint() { session.requestHint(selected: selectedPosition); checkWinState() }

    private func checkWinState() {
        if session.completed { showWinScreen = true }
    }
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| @Observable not triggering updates | M | H | Mutate session then read properties in ViewModel methods; test AC-2 |
| Win screen shown prematurely | M | H | `checkWinState` only on `session.completed` (validator Rule 5) |
| Logic duplicated from session | M | M | FR-11: ViewModel delegates all mutations |
| selectedPosition stale after reset | L | M | Clear on `reset()` (FR-8) |
