## 1. Problem Statement

When a player satisfies all puzzle rules (Formal Rules Rule 5), the game must celebrate completion and offer a path to play again. Without a win screen, the playable loop ends silently — there is no feedback that the puzzle is solved, and the board remains editable. GDD specifies a minimal win overlay for v0.1: title + Play Again, with the board locked until reset.

## 2. Goals

- Show a win overlay when `GameViewModel.showWinScreen == true`.
- Display "Puzzle Complete!" title per GDD.
- Provide a **Play Again** button that resets the puzzle (P3.5).
- Lock the board (no cell edits, disabled toolbar) while the win overlay is showing or after completion.
- Keep v0.1 minimal: no elapsed time or hints-used display (deferred to v1.0).

## 3. Non-Goals

- Elapsed time display (v1.0, P4.6).
- Hints-used display (v1.0).
- Next puzzle navigation (v1.1, P4.8).
- Win animations (P5.2).
- Confetti or elaborate celebration (polish phase).
- Share score / social features.

## 4. Personas & User Stories

- **As a player**, I want to see "Puzzle Complete!" when I solve the puzzle so I feel rewarded.
- **As a player**, I want a Play Again button so I can retry the same puzzle.
- **As a player**, I want the board locked after winning so I don't accidentally change my completed solution.
- **As a developer**, I want the win screen decoupled from `GameView` as a reusable overlay component.

## 5. Functional Requirements

- **FR-1.** `WinScreen` MUST display the title **"Puzzle Complete!"** (GDD Win Screen table).
- **FR-2.** `WinScreen` MUST provide a **Play Again** primary button.
- **FR-3.** Tapping Play Again MUST call `onPlayAgain` callback, which triggers `viewModel.reset()` (P3.5).
- **FR-4.** After Play Again, the win overlay MUST dismiss and the board MUST return to empty initial state.
- **FR-5.** While `isCompleted == true`, the board MUST reject all cell taps (Formal Rules §Placement Validation).
- **FR-6.** While `isCompleted == true`, toolbar buttons (Undo, Redo, Reset, Hint) and mode toggle MUST be disabled (P3.12, P3.11).
- **FR-7.** The win overlay MUST appear only when `validationResult.isComplete == true` (not on partial or invalid full boards).
- **FR-8.** `WinScreen` MUST NOT display elapsed time or hints used in v0.1 (GDD).
- **FR-9.** Play Again button MUST have VoiceOver label "Play again" and meet ≥ 44×44 pt touch target.
- **FR-10.** Win overlay MUST be dismissible only via Play Again (no swipe-to-dismiss if presented as sheet).

## 6. Non-Functional Requirements

- **Performance** — Instant overlay presentation; no heavy animation in v0.1.
- **Accessibility** — Title announced by VoiceOver; Play Again button focusable; respects Reduce Motion (static overlay).
- **Maintainability** — `UI/Screens/WinScreen.swift`; callback-based, no direct ViewModel dependency.
- **Internationalization** — Title and button text via string literals acceptable for v0.1.

## 7. Acceptance Criteria

- **AC-1.** *Given* a board matching `puzzle.solution`, *when* the last animal is placed, *then* the win overlay appears with "Puzzle Complete!".
- **AC-2.** *Given* the win overlay is showing, *when* the player taps Play Again, *then* the overlay dismisses and the board is empty.
- **AC-3.** *Given* a completed board, *when* the player taps a cell, *then* no state change occurs (board locked).
- **AC-4.** *Given* a full board with rule violations, *when* validated, *then* the win overlay does NOT appear.
- **AC-5.** *Given* a partial board (not all animals placed), *when* validated, *then* the win overlay does NOT appear.
- **AC-6.** *Given* VoiceOver, *when* the win overlay appears, *then* "Puzzle Complete!" is announced.
- **AC-7.** *Given* the win overlay, *when* Play Again is tapped, *then* `hintsUsed == 0`, undo/redo stacks empty, `inputMode == .place` (P3.5).

## 8. Data Model

Presentation-only:

```swift
// UI/Screens/WinScreen.swift
struct WinScreen: View {
    var onPlayAgain: () -> Void = {}

    var body: some View {
        VStack(spacing: AppSpacing.lg) {
            Text("Puzzle Complete!")
                .font(AppTypography.largeTitle)
                .accessibilityAddTraits(.isHeader)

            Button("Play Again", action: onPlayAgain)
                .buttonStyle(.borderedProminent)
                .accessibilityIdentifier("playAgainButton")
        }
        .padding(AppSpacing.xl)
    }
}
```

Win trigger in `GameViewModel` (P3.13):

```swift
private func checkWinState() {
    if session.completed && session.validationResult.isComplete {
        showWinScreen = true
    }
}
```

Board lock in `GameViewModel.handleCellTap`:

```swift
guard !isCompleted else { return }
```

## 10. UI / UX

**Overlay layout:**

```
┌─────────────────────────┐
│                         │
│    (dimmed board        │
│     behind overlay)     │
│                         │
│   ┌─────────────────┐   │
│   │ Puzzle Complete!│   │
│   │                 │   │
│   │  [ Play Again ] │   │
│   └─────────────────┘   │
│                         │
└─────────────────────────┘
```

**Visual design:**

- Centered card with rounded corners (P1.4 `AppRadius.lg`).
- Semi-transparent backdrop dimming the board.
- Prominent "Play Again" button using accent color.
- No stats text in v0.1.

**Post-win board state:**

- Cells remain showing the completed solution (visual proof).
- All interaction disabled until Play Again.
- Toolbar grayed out.

**GDD Win Screen spec (v0.1):**

| Element | v0.1 |
|---------|------|
| Title | "Puzzle Complete!" |
| Elapsed time | Hidden |
| Hints used | Hidden |
| Primary action | Play Again |
| Board after win | Locked |

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Win shown on invalid full board | M | H | FR-7: only `isComplete`, not merely full |
| Play Again doesn't fully reset | M | H | AC-7 verifies all counters/stacks |
| Overlay blocks VoiceOver navigation to board | L | L | Expected; board is locked |
| Sheet swipe dismisses without reset | M | M | FR-10: `.interactiveDismissDisabled()` |

## 15. Rollout Plan

- No feature flag. Part of v0.1 playable milestone.
- Merge when AC-1–AC-7 pass.
- Rollback: remove overlay; win silently locks board (degraded UX).

## 16. Test Plan

- **Unit** — Win trigger in ViewModel: solution board → `showWinScreen == true`; invalid full board → false (P3.13 tests).
- **UI test (T-082)** — Complete puzzle → assert `playAgainButton` visible → tap → assert board empty.
- **Manual** — Solve puzzle manually; verify overlay; tap Play Again; verify reset; attempt cell tap on locked board.
- **Accessibility** — VoiceOver announces title (AC-6); Play Again focusable.
