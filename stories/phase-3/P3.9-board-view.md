## 1. Problem Statement

Players need to see the full 8×8 puzzle grid with colored regions that partition the board. `CellView` (P3.8) renders individual cells, but there is no layout component that arranges 64 cells into a responsive grid, paints region background colors from puzzle data, and forwards taps to the game layer. Without `BoardView`, the primary game screen cannot be composed.

## 2. Goals

- Render an N×N grid (8×8 for MVP) using `CellView` for each cell.
- Paint each cell's background with its region color from `puzzle.regions[].color`.
- Size the board responsively so it fits iPhone SE through Pro Max in portrait.
- Forward cell taps and selection state to the parent (`GameViewModel` in P3.13).
- Compute and pass violation highlights per cell from `ValidationResult`.

## 3. Non-Goals

- Colorblind region patterns (P3.10) — this story uses solid region colors only; patterns overlay in P3.10.
- Toolbar, mode toggle, or win overlay (P3.11–P3.15).
- Game logic / `GameSession` mutations (P3.2, wired via P3.13).
- Animations on placement (P5.1).

## 4. Personas & User Stories

- **As a player**, I want to see the whole board at once so I can reason about rows, columns, and regions.
- **As a player**, I want region colors to make irregular region shapes obvious at a glance.
- **As a player on a small phone**, I want the board to fit on screen without horizontal scrolling.
- **As a developer**, I want a parameterized `BoardView` that takes puzzle geometry and cell states without importing the engine.

## 5. Functional Requirements

- **FR-1.** `BoardView` MUST render a grid of `size × size` cells where `size` comes from the puzzle (8 for MVP).
- **FR-2.** Each cell MUST display the background color of its `regionId` using the hex color from `puzzle.regions` (Roadmap Decision Log #9: colors assigned in puzzle JSON).
- **FR-3.** Each cell MUST be rendered via `CellView` (P3.8) with correct `row`, `col`, `regionId`, `state`, `isSelected`, and `isViolating`.
- **FR-4.** `isViolating` MUST be `true` when the cell's `Position` appears in any `RuleViolation.positions` from the supplied `ValidationResult` (P2.7).
- **FR-5.** Tapping a cell MUST invoke an `onCellTap(Position)` callback to the parent; `BoardView` MUST NOT call `GameSession` directly.
- **FR-6.** The board MUST fit within the available width on iPhone SE (375 pt) through Pro Max (430 pt) in portrait, maintaining square cells.
- **FR-7.** Inter-cell spacing and outer padding MUST use design tokens from P1.4 (`AppSpacing`).
- **FR-8.** The board MUST use `LazyVGrid` or equivalent layout that scales cell size based on available geometry (e.g. `GeometryReader`).
- **FR-9.** When `isBoardLocked == true` (puzzle complete), taps SHOULD still be delivered but the parent/session rejects mutations (P3.2 FR-10); optional visual dimming is nice-to-have.

## 6. Non-Functional Requirements

- **Performance** — 64 `CellView` instances at 60 fps; avoid per-frame layout recalculation.
- **Accessibility** — Grid announced as a single accessibility container; individual cells retain VoiceOver from `CellView` (P3.8 FR-7).
- **Maintainability** — SwiftUI struct in `UI/Components/`; SwiftUI previews with sample puzzle from P2.8.
- **Internationalization** — No user-facing strings in this component.
- **Scalability** — Layout algorithm MUST work for any `size` value, not hard-coded to 8 (future puzzles).

## 7. Acceptance Criteria

- **AC-1.** *Given* an 8×8 puzzle, *when* `BoardView` renders, *then* exactly 64 cells appear in a square grid.
- **AC-2.** *Given* two adjacent cells in different regions, *when* rendered, *then* their background colors differ per puzzle JSON.
- **AC-3.** *Given* a `ValidationResult` with a row violation at positions A and B, *when* rendered, *then* both cells show `isViolating == true`.
- **AC-4.** *Given* a tap on cell (row 3, col 5), *when* the tap completes, *then* `onCellTap(Position(row: 3, col: 5))` is called once.
- **AC-5.** *Given* iPhone SE simulator in portrait, *when* the board is measured, *then* the full grid fits without horizontal scroll and each cell touch target ≥ 44 pt (via P3.8).
- **AC-6.** *Given* `selectedPosition == Position(row: 1, col: 2)`, *when* rendered, *then* only that cell shows the selection ring.

## 8. Data Model

Presentation inputs (no persistence):

```swift
// UI/Components/BoardView.swift
struct BoardView: View {
    let puzzle: Puzzle
    let cells: [Cell]                    // row-major, length == size*size
    let validationResult: ValidationResult
  var selectedPosition: Position? = nil
  var isBoardLocked: Bool = false
  var animalIcon: Image                  // from theme; placeholder until P4.1
  var onCellTap: (Position) -> Void = { _ in }

    private func isViolating(at position: Position) -> Bool {
        validationResult.violations.contains { $0.positions.contains(position) }
    }
}
```

Helper for region color lookup:

```swift
// UI/Components/RegionColorMap.swift (optional)
struct RegionColorMap {
    init(regions: [Region])
    func color(for regionId: Int) -> Color  // hex → SwiftUI Color
}
```

## 10. UI / UX

**Layout flow:**

1. `GeometryReader` measures available width.
2. Compute `cellSize = (width - padding - gaps) / size`.
3. `LazyVGrid` with `size` flexible columns of `cellSize`.
4. Each cell: region color background → `CellView` overlay.

**Visual hierarchy:**

| Layer | Content |
|-------|---------|
| Background | Region hex color per cell |
| Foreground | `CellView` (X, animal, selection, violation) |
| Grid lines | Optional 1 pt separator between cells (token from P1.4) |

**Empty / loading / error states:**

- Loading: parent (`GameView`) shows progress while puzzle loads (P3.16); `BoardView` assumes puzzle is ready.
- Error: not handled here; `GameView` shows error if puzzle fails to load.

**Previews:** `#Preview` with P2.8 MVP puzzle, mixed cell states, and a sample violation.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Board overflows on SE | M | H | Geometry-based sizing; AC-5 manual test on SE |
| Region color parse failure | L | M | Validate hex at puzzle load (P2.3); fallback gray |
| Violation highlight misses diagonal pair | M | M | Use `RuleViolation.positions` from validator, not ad-hoc logic |
| Cell size below 44 pt | M | H | Enforce minimum via layout math; parent may reduce padding |
