## 1. Problem Statement

Formal Rules defines two mutually exclusive input modes — **Place** (default) and **Mark** — that change how cell taps are interpreted. Without a visible toggle, players cannot mark cells as impossible (X) or switch back to placing animals. `GameSession` holds `inputMode` (P3.1) but the UI has no control to change it.

## 2. Goals

- Provide a toolbar control to switch between Place and Mark modes.
- Reflect the current mode visually so players always know what the next tap will do.
- Wire toggle to `GameSession.inputMode` via `GameViewModel` (P3.13).
- Default to Place mode on session init and after reset (P3.5).

## 3. Non-Goals

- Gesture-based mode switching (e.g. long-press) — toolbar toggle only for v0.1.
- Per-mode onboarding tooltips (tutorial is v1.1).
- Undo/redo/reset/hint buttons (P3.12).
- Changing tap behavior logic (implemented in P3.2 / `GameViewModel` routing).

## 4. Personas & User Stories

- **As a player**, I want to switch to Mark mode so I can mark cells as impossible with an X.
- **As a player**, I want to see which mode is active so I don't accidentally place an animal when I meant to mark.
- **As a player**, I want Place mode as the default so the most common action is one tap away.
- **As a developer**, I want a small reusable `InputModeToggle` component bound to `InputMode`.

## 5. Functional Requirements

- **FR-1.** The UI MUST expose a toggle between `InputMode.place` and `InputMode.mark`.
- **FR-2.** Default mode on new session and after reset MUST be `.place` (Formal Rules).
- **FR-3.** Tapping the toggle MUST update `GameSession.inputMode` and refresh the UI.
- **FR-4.** The active mode MUST be visually indicated (selected segment, icon highlight, or label).
- **FR-5.** In **Place** mode, cell taps MUST route to place/remove actions per Formal Rules Interaction Model (P3.2).
- **FR-6.** In **Mark** mode, cell taps MUST route to toggle-blocked actions per Formal Rules (P3.2).
- **FR-7.** Mode toggle MUST remain enabled during play; MUST be disabled when board is locked by completion (P3.15).
- **FR-8.** VoiceOver MUST announce the current mode when the toggle is focused and when it changes.

## 6. Non-Functional Requirements

- **Performance** — Instant mode switch; no async work.
- **Accessibility** — Segmented control or two buttons with clear labels ("Place mode", "Mark mode"); minimum 44×44 pt touch targets (P1.4).
- **Maintainability** — `UI/Components/InputModeToggle.swift`; binds to `Binding<InputMode>` or callback.
- **Internationalization** — Labels use `LocalizedStringKey` placeholders for v1.1; English strings acceptable for v0.1.

## 7. Acceptance Criteria

- **AC-1.** *Given* a fresh game, *when* the screen loads, *then* Place mode is selected and indicated visually.
- **AC-2.** *Given* Place mode active, *when* the player taps the Mark segment, *then* `inputMode == .mark` and the UI updates.
- **AC-3.** *Given* Mark mode active, *when* the player taps an empty cell, *then* the cell becomes blocked (X) via P3.2 routing.
- **AC-4.** *Given* Place mode active, *when* the player taps an empty cell, *then* an animal is placed (P3.2).
- **AC-5.** *Given* a completed puzzle (board locked), *when* the toggle is shown, *then* it is disabled.
- **AC-6.** *Given* VoiceOver focused on the toggle, *when* mode changes, *then* the new mode is announced.

## 8. Data Model

Uses `InputMode` from P3.1:

```swift
// Engine/GameSession.swift (P3.1)
enum InputMode: Equatable {
    case place
    case mark
}
```

UI component:

```swift
// UI/Components/InputModeToggle.swift
struct InputModeToggle: View {
    @Binding var mode: InputMode
    var isEnabled: Bool = true

    var body: some View {
        Picker("Input mode", selection: $mode) {
            Text("Place").tag(InputMode.place)
            Text("Mark").tag(InputMode.mark)
        }
        .pickerStyle(.segmented)
        .disabled(!isEnabled)
    }
}
```

Formal Rules tap routing reference:

| Mode | Tap empty | Tap blocked (X) | Tap animal |
|------|-----------|-----------------|------------|
| **Place** | Place animal | No effect | Remove animal |
| **Mark** | Toggle blocked (X) | Clear X → empty | No effect |

## 9. API Surface

N/A — SwiftUI component. Binding:

```swift
InputModeToggle(mode: $viewModel.inputMode, isEnabled: !viewModel.isCompleted)
```

## 10. UI / UX

**Placement:** Above the board or in the toolbar row, below the navigation title — consistent with GDD Game screen layout.

**Visual design:**

- Segmented control: `[ Place | Mark ]` with accent fill on active segment.
- Optional icons: pawprint for Place, `xmark` for Mark (SF Symbols).
- Subtle mode hint text below toggle (nice-to-have): "Tap cells to place animals" / "Tap cells to mark impossible".

**States:**

| State | Appearance |
|-------|------------|
| Place active | Left segment highlighted |
| Mark active | Right segment highlighted |
| Disabled (win) | Grayed out, non-interactive |
