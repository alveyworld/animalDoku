## 1. Problem Statement

Players occasionally need assistance without spoiling the entire puzzle. Formal Rules defines hint behavior precisely: reveal one correct cell from `puzzle.solution`, max 3 per attempt, undoable, blocked on non-empty targets. The Roadmap previously marked hints as blocked on product decision; that decision is now resolved in Formal Rules § Hint System.

## 2. Goals

- Implement `HintService` (or equivalent) that selects a target cell and reveals the solution placement.
- Enforce max 3 hints per puzzle attempt via `hintsUsed`.
- Record hint placements as `GameAction.hint(position)` for undo/redo (P3.3/P3.4).
- Respect preconditions: target must be empty (not animal, not blocked).

## 3. Non-Goals

- Highlight-only or rule-explanation hints (not in spec).
- Unlimited hints or paid hint packs (monetization open question).
- Hint UI beyond toolbar button (P3.12); no hint count display on win screen in v0.1 (GDD).

## 4. Personas & User Stories

- **As a player**, I want a hint to place one correct animal so I can get unstuck while still solving the rest myself.
- **As a player**, I want hints limited to 3 so the puzzle retains challenge.
- **As a player**, I want to undo a hint if I tapped it by mistake.
- **As a developer**, I want hint logic isolated in `HintService` for testability.

## 5. Functional Requirements

- **FR-1.** A hint MUST reveal the correct animal placement for one unsolved cell from `puzzle.solution` (set cell to `.animal`).
- **FR-2.** Target selection: if the player has a cell selected, hint that cell; otherwise hint the **first unsolved cell in row-major order** (row 0 col 0 → row 0 col 1 → …).
- **FR-3.** Maximum **3** hints per puzzle attempt (`hintsUsed` MUST NOT exceed 3).
- **FR-4.** Target cell MUST be `.empty`; if `.animal` or `.blocked`, the hint MUST NOT apply (no-op per AC-H.3).
- **FR-5.** On success, `hintsUsed` MUST increment by 1 and a `hint(position)` action MUST be pushed to the undo stack.
- **FR-6.** Hint placements MUST be undoable (P3.3) and redoable (P3.4).
- **FR-7.** Hints MUST be rejected when `hintsUsed == 3` (no-op per AC-H.2).
- **FR-8.** Hints MUST be rejected when the board is locked by completion.
- **FR-9.** After a successful hint, `validationResult` MUST be recomputed (may trigger completion if last animal placed).

## 6. Non-Functional Requirements

- **Performance** — Target scan is O(n²) worst case; trivial at 8×8.
- **Security** — Solution lives in bundled JSON; no network exposure.
- **Maintainability** — `Engine/HintService.swift` separate from `GameSession`; session delegates to service.
- **Reliability** — Hint always places the solution position's animal, never a wrong cell.

## 7. Acceptance Criteria

- **AC-1 (AC-H.1).** *Given* `hintsUsed == 0` and an empty target in the solution, *when* hint is requested, *then* that cell becomes `.animal` and `hintsUsed == 1`.
- **AC-2 (AC-H.2).** *Given* `hintsUsed == 3`, *when* hint is requested, *then* no change occurs.
- **AC-3 (AC-H.3).** *Given* a blocked target cell, *when* hint is requested, *then* no change occurs.
- **AC-4.** *Given* a selected empty cell that is in the solution, *when* hint is requested, *then* that specific cell is revealed.
- **AC-5.** *Given* no selection and multiple empty cells, *when* hint is requested, *then* the first empty cell in row-major order that is in the solution is revealed.
- **AC-6.** *Given* a successful hint, *when* `undo()`, *then* the cell is `.empty` and `hintsUsed` decrements.
- **AC-7.** *Given* a completed board, *when* hint is requested, *then* no change occurs.

## 8. Data Model

```swift
// Engine/HintService.swift
struct HintService {
    static let maxHints = 3

    func targetPosition(
        board: [Cell],
        solution: [Position],
        selected: Position?
    ) -> Position? { /* FR-2, FR-4 */ }

    func canHint(hintsUsed: Int, board: [Cell], target: Position?) -> Bool
}

extension GameSession {
  var canHint: Bool { hintsUsed < HintService.maxHints && !completed /* + target exists */ }
  func requestHint(selected: Position? = nil) { /* delegate to HintService; record action */ }
}
```

`GameAction.hint(Position)` already defined in P2.2.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Hint on blocked cell not rejected | M | M | AC-3 / FR-4 test |
| Row-major scan picks wrong cell | L | H | AC-5 deterministic test with known board |
| Hint completes puzzle unexpectedly | M | L | Expected behavior; win flow (P3.15) handles |
| Counter exceeds 3 | L | M | Guard in `canHint` and `requestHint` |
