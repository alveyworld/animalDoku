## 1. Problem Statement

Blocked cells today use a thin stroked diagonal **X** (`BlockedMark` in `CellView`) tinted with the primary/icon color. On brighter region fills (P8.1) that mark can look small, low-emphasis, or hard to scan. Players need marks that read instantly as “ruled out” — large, bold, and high-contrast against the cell.

## 2. Goals

- Replace the thin path-based X with a **white typographic X**.
- Use a **fat, rounded sans-serif** style (heavy weight, rounded terminals).
- Size the glyph so it **fills most of the cell** (dominant visual, not a small inset icon).
- Keep VoiceOver / state semantics unchanged (`blocked` / Mark / Clear mark).

## 3. Non-Goals

- Changing mark gesture behavior (still single-tap / drag-to-mark from P6.5 / P5.7).
- Custom drawn vector assets unless the system font can’t hit the look.
- Recoloring animal icons in this story (only the blocked mark).
- Per-theme mark colors (white is fixed for contrast on bright fills).

## 4. Personas & User Stories

- **As a player**, I want large white Xs so that marked cells are obvious at a glance.
- **As a player on a bright board**, I want the mark to stay readable on every region color.
- **As a developer**, I want mark styling centralized in `BlockedMark` (or a small design token) so weight/size aren’t scattered.

## 5. Functional Requirements

- **FR-1.** A blocked cell MUST render a white **“X”** (or equivalent glyph), not the current thin dual-stroke Path alone.
- **FR-2.** The mark MUST use a **bold / black (heavy) rounded sans-serif** treatment — prefer a system rounded design (e.g. SF Rounded) at `.black` / `.heavy` weight, or an approved bundled font if SF Rounded is insufficient.
- **FR-3.** The glyph MUST **fill most of the square** — target roughly **95%** of the cell’s min(width, height) as font size / layout diameter, nearly touching the cell corners.
- **FR-4.** Mark color MUST be **white** (`Color.white`) in default and high-contrast modes unless measurement shows a specific region fails contrast — then document an exception (still prefer white on the P8.1 palette).
- **FR-5.** Placement / remove animations (P5.1) MUST still apply to the mark appear/disappear transition.
- **FR-6.** Accessibility labels and custom actions MUST remain correct; the visual change MUST NOT rely on color alone for meaning (shape is still an X).

## 6. Non-Functional Requirements

- **Performance** — Text glyph is cheap; no frame impact vs Path.
- **Accessibility** — X remains a clear shape; white on dark gray / blue / vermillion must stay visible (verify on all 8 region fills).
- **Maintainability** — Update `BlockedMark` in `UI/Components/CellView.swift` (or extract if it grows); optional `AppTypography` / mark token for size scale.
- **Reliability** — Dynamic Type does **not** need to scale board marks (board cells are fixed layout); keep mark size tied to cell geometry, not content-size category.

## 7. Acceptance Criteria

- **AC-1.** *Given* a blocked cell, *when* rendered, *then* the mark is a white X (not gray/primary thin strokes).
- **AC-2.** *Given* the mark, *when* inspected, *then* it uses a heavy rounded sans-serif style.
- **AC-3.** *Given* cell size S, *when* laid out, *then* the X visually occupies ~95% of S, nearly touching the corners.
- **AC-4.** *Given* each region fill from P8.1 (or current palette), *when* a mark is shown, *then* the X remains clearly visible.
- **AC-5.** *Given* existing CellView / a11y tests, *when* updated, *then* blocked-state coverage still passes.

## 8. Data Model

No model changes. Presentation only:

```swift
// Conceptual — replace Path-based BlockedMark
struct BlockedMark: View {
    var body: some View {
        Text("×") // or "X"
            .font(.system(size: cellFontSize, weight: .black, design: .rounded))
            .foregroundStyle(.white)
            .minimumScaleFactor(0.5)
            .lineLimit(1)
    }
}
```

Font size derived from `GeometryReader` / cell frame (not a fixed pt that breaks on SE vs Pro Max).

## 10. UI / UX

- Mark: white, heavy, rounded, nearly full-bleed in the cell.
- Prefer multiplication sign `×` (U+00D7) if it looks more balanced in SF Rounded; otherwise capital `X`. Pick one in implementation and lock it in tests/previews.
- No drop shadow required; optional very subtle shadow only if white-on-yellow fails AC-4 (prefer avoiding chrome noise).
- Animal icons stay as today; only blocked state changes.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| White X weak on yellow (`#F0E442`) | M | M | Slightly larger glyph; optional soft dark halo only if needed; verify AC-4 |
| Font size clips cell border | L | L | Cap at ~85% + padding |
| `×` vs `X` metrics differ by OS | L | L | Lock one glyph; snapshot/manual check on target iOS |
