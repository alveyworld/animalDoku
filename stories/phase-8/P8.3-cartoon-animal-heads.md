## 1. Problem Statement

Placed animals currently render as flat theme icons (SF Symbol fallbacks or simple asset catalog glyphs) that sit statically in the cell. The desired look is **friendly cartoon animal heads** (frogs, dogs, foxes) matching the provided examples, with a light **look-around** animation after placement so the board feels alive. Without new art and a small animation system, themes stay generic and miss the Phase 8 personality goal.

## 2. Goals

- Replace board / theme animal glyphs with **cartoon head** artwork for Frogs, Dogs, and Foxes, styled like the reference images (flat color, bold black outlines, big eyes, white background cropped/transparent in-cell).
- After an animal is **placed**, animate the head so its gaze **looks around the board** (pupils / eyes shift), then settle.
- Honor **Reduce Motion** (and `-uiTestReduceMotion`): skip or shorten look-around to an instant static pose.
- Keep themes cosmetic only — no effect on puzzle logic.
- Update theme picker / settings previews to use the same heads for consistency.

## 3. Non-Goals

- Full-body animals, walk cycles, or physics.
- New themes beyond Frogs / Dogs / Foxes.
- Continuous forever-idle animation on every placed animal (optional later); this story focuses on **post-place** look-around (plus optional rare idle if cheap).
- Lip-sync, blink-as-a-service, or randomized personality per cell beyond gaze.
- Changing mark Xs (P8.2) or region palette (P8.1).

## 4. Personas & User Stories

- **As a player**, I want cute cartoon animal heads so that placing a piece feels delightful.
- **As a player**, I want the animal to glance around after I place it so the board feels responsive and fun.
- **As a player with Reduce Motion**, I want a static head with no eye movement.
- **As a developer**, I want a reusable `AnimalHeadView` so CellView, theme picker, and win chrome share one component.

## 5. Functional Requirements

- **FR-1.** Each theme (`frogs`, `dogs`, `foxes`) MUST resolve to a cartoon head asset (or composited layers) suitable for an 8×8 cell, matching the reference style.
- **FR-2.** `CellView` animal state MUST render the themed cartoon head (not SF Symbol paw/hare fallbacks) when assets are present.
- **FR-3.** After a successful **place** into a cell, the head MUST play a **look-around** animation: eyes/pupils move through at least 2–3 gaze directions (e.g. left → right → center, or toward board center), over roughly **0.8–1.5 s**, then rest at a neutral forward gaze.
- **FR-4.** Look-around MUST NOT run on load of an already-saved board full of animals (resume mid-puzzle) unless product later asks for idle; default: **only on the place action that created that animal** (and optionally on Play Again fresh places).
- **FR-5.** When Reduce Motion is active (system or UI-test force), look-around MUST be skipped; head appears in neutral pose.
- **FR-6.** Removing an animal MUST cancel any in-flight look-around for that cell.
- **FR-7.** Theme switcher / settings theme previews MUST show the same cartoon heads.
- **FR-8.** Heads MUST remain readable on P8.1 region fills (scale ~fill cell with small padding, similar to today’s animal icon padding).

## 6. Non-Functional Requirements

- **Performance** — Prefer compositing 1–2 static images + tiny pupil offsets over large frame flipbooks; 60 fps on SE-class; no animation work for empty/blocked cells.
- **Accessibility** — VoiceOver labels unchanged (“animal”); animation is decorative; Reduce Motion respected via existing `Motion` helpers.
- **Maintainability** — `AnimalHeadView(theme:gaze:)` (or similar) owns art + gaze; `CellView` passes theme + animation trigger.
- **Reliability** — Missing asset falls back gracefully (keep SF Symbol) without crashing.
- **Art pipeline** — Export @2x/@3x PNGs with transparent backgrounds into `Assets.xcassets/Themes/`.

## 7. Acceptance Criteria

- **AC-1.** *Given* Frogs theme, *when* an animal is placed, *then* the cell shows a cartoon frog head in the reference style (not a generic leaf/SF glyph).
- **AC-2.** *Given* Dogs / Foxes themes, *when* placed, *then* the matching cartoon head appears.
- **AC-3.** *Given* a fresh place, *when* the place animation completes (or concurrently), *then* the head looks around (visible eye/pupil motion) and returns to center.
- **AC-4.** *Given* Reduce Motion on, *when* placing, *then* no look-around motion occurs.
- **AC-5.** *Given* theme picker, *when* browsing themes, *then* previews use the cartoon heads.
- **AC-6.** *Given* a resumed save with existing animals, *when* the board loads, *then* heads are static (no mass look-around).

## 8. Data Model

No puzzle-schema change. Presentation/animation state only:

```swift
struct AnimalGaze: Equatable {
    var offset: CGSize // pupil translation in points, clamped to eye bounds
}

enum AnimalHeadPose {
    case neutral
    case looking(AnimalGaze)
}
```

Optional: one-shot `lookAroundToken` / `placedAt` on the cell UI layer (not persisted in `SaveGame`).

## 10. UI / UX

### Art direction (from references)

| Theme | Reference | Notes |
|-------|-----------|--------|
| Frogs | `assets/frog-head-example.png` | Green head, big dome eyes, pink tongue smile |
| Dogs | `assets/dog-head-example.png` | Tan face, brown floppy ears, pink tongue |
| Foxes | `assets/fox-head-example.png` | Orange face, cream muzzle, black ear tips |

Shared style: flat fills, thick black outlines, oversized circular eyes with pupils + specular highlight, cheerful expression, head-only (no body).

### Animation

- Trigger: after place (and after place transition if both run — stagger so look-around starts as the head pops in).
- Motion: pupils (or whole-eye layer) ease to offsets; 2–3 keyframes; settle neutral.
- Recommended implementation: **layered assets** — base head (eyes white, no pupils or empty sockets) + pupil layer(s) offset in SwiftUI. Alternative: single sprite + subtle rotation/tilt of the whole head if layered art isn’t ready (weaker match to “look around”).
- Do not animate blocked marks or empty cells.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Single flat PNG can’t move pupils | H | M | Export layered base + pupils, or approximate with slight head tilt |
| Look-around on every cell feels noisy | M | M | Trigger only on place; short duration; no loop |
| Busy board + many simultaneous animations | L | M | Only the newly placed cell animates |
| Asset size / memory | L | L | Shared imagesets; no per-cell bitmap copies |
| Reduce Motion missed | M | H | Reuse `Motion` / `forceReduceMotion` paths; UITest coverage |
