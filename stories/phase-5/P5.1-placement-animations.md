## 1. Problem Statement

The GDD's identity is calm, polished, satisfying gameplay with gentle animations. Today, placing or removing an animal snaps instantly with no motion, which feels abrupt. Gentle place/remove animations reinforce the relaxing feel — but they must respect the system Reduce Motion setting so motion-sensitive players get a static experience.

## 2. Goals

- Add subtle animations for placing and removing an animal (e.g. soft scale/fade).
- Keep animations short and calm, consistent with the GDD aesthetic.
- Fully respect the system Reduce Motion setting.
- Ensure animations never block input or delay validation feedback.

## 3. Non-Goals

- Win celebration animation (P5.2).
- Haptic feedback (P5.6).
- Region/board transition animations beyond cell state changes.
- New design tokens (reuse P1.4).

## 4. Personas & User Stories

- **As a player**, I want gentle motion when I place an animal so that the game feels polished.
- **As a motion-sensitive player**, I want animations disabled when Reduce Motion is on.
- **As a developer**, I want a single animation policy so cells animate consistently.

## 5. Functional Requirements

- **FR-1.** Placing an animal MUST play a gentle appearance animation (e.g. scale-in + fade).
- **FR-2.** Removing an animal MUST play a gentle disappearance animation.
- **FR-3.** When Reduce Motion is enabled, state changes MUST apply instantly (no scale/translate); a cross-fade at most.
- **FR-4.** Animations MUST NOT block subsequent taps or delay validation/violation highlighting.
- **FR-5.** Animation duration and curve MUST be centralized (reused by P5.2).

## 6. Non-Functional Requirements

- **Performance** — Maintain 60fps on the oldest supported device; animations are GPU-light.
- **Accessibility** — Honors `accessibilityReduceMotion`; motion is never required to understand state.
- **Privacy** — N/A.
- **Maintainability** — Animation constants in one place; cells opt in via a shared modifier.
- **Reliability** — Rapid place/remove sequences do not leave cells in a mid-animation stuck state.

## 7. Acceptance Criteria

- **AC-1.** *Given* Reduce Motion off, *when* an animal is placed, *then* it animates in gently (~150–250ms).
- **AC-2.** *Given* Reduce Motion on, *when* an animal is placed, *then* it appears immediately with no scale/translate.
- **AC-3.** *Given* rapid taps placing/removing, *when* animating, *then* no visual glitch or stuck state remains.
- **AC-4.** *Given* an invalid placement, *when* placed, *then* violation highlighting still appears without animation-induced delay.

## 8. Data Model

No new model. Uses `CellState` transitions (Formal Rules) and design tokens (P1.4).

```swift
// UI/DesignSystem/Motion.swift (or extend tokens)
enum Motion {
    static let placeDuration: Double  // ~0.2s
    static let curve: Animation
    static func respectingReduceMotion(_ reduce: Bool) -> Animation?
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Reduce Motion not honored | M | H | Read environment flag; test both states |
| Animations feel sluggish | M | M | Keep short (<250ms), test on device |
| Stuck mid-animation on rapid taps | L | M | Use value-driven transitions, not manual timers |

## 15. Rollout Plan

- No feature flag. Ships in v1.0 polish pass.
- Rollback: disable animation modifier; instant state changes remain functional.

## 16. Test Plan

- **Manual** — Feel test on device for calmness; rapid-tap stress.
