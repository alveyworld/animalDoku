## 1. Problem Statement

Placement and win moments have visual and audio feedback (P4.5, P5.1, P5.2) but no tactile reinforcement. Subtle haptics can make actions feel more satisfying on supported devices. Because haptics are optional polish and listed as Future in the roadmap, this ships in v1.1 with a user-toggle or respect for system settings so it never annoys.

## 2. Goals

- Provide light haptic feedback on place, remove, and win events.
- Keep haptics subtle and aligned with the calm GDD tone.
- Make haptics optional (settings toggle or off when sound disabled — product choice in Open Questions).
- No-op gracefully on devices/simulators without haptic hardware.

## 3. Non-Goals

- Heavy or repeated vibration patterns.
- Haptics on every UI tap (toolbar, settings).
- Custom Core Haptics compositions.
- Replacing visual or audio feedback.

## 4. Personas & User Stories

- **As a player**, I want a subtle tap feel when placing an animal so that actions feel tactile.
- **As a player who dislikes vibration**, I want to disable haptics.
- **As a developer**, I want a small `HapticService` mirroring `SoundService` patterns.

## 5. Functional Requirements

- **FR-1.** The app SHOULD trigger a light impact haptic on successful animal placement.
- **FR-2.** The app MAY trigger a softer haptic on remove (optional; lighter than place).
- **FR-3.** The app SHOULD trigger a success/notification haptic on puzzle completion.
- **FR-4.** Haptics MUST be suppressible via a user preference (new `hapticsEnabled` in SettingsStore or tied to sound toggle — see Open Questions).
- **FR-5.** Haptics MUST be a no-op on Simulator and devices without Taptic Engine.
- **FR-6.** Haptics MUST NOT fire when the app is backgrounded.

## 6. Non-Functional Requirements

- **Performance** — `UIImpactFeedbackGenerator` prepared once; negligible latency.
- **Accessibility** — Haptics supplement visual feedback; never required to play.
- **Privacy** — No data collection.
- **Battery** — Minimal; only on discrete game events, not continuous.
- **Maintainability** — Central `HapticService` enum mirroring `GameSound`.

## 7. Acceptance Criteria

- **AC-1.** *Given* haptics enabled on a supported device, *when* an animal is placed, *then* a light haptic is felt once.
- **AC-2.** *Given* haptics disabled, *when* any action occurs, *then* no haptic fires.
- **AC-3.** *Given* puzzle completion, *when* win triggers, *then* a success-style haptic fires once (if enabled).
- **AC-4.** *Given* Simulator, *when* actions occur, *then* no crash or error.
- **AC-5.** *Given* rapid place/remove, *when* haptics fire, *then* they do not queue into a long vibration burst.

## 8. Data Model

```swift
enum GameHaptic { case place, remove, win }

final class HapticService {
    init(settings: SettingsStore)
    func play(_ haptic: GameHaptic)  // no-op when disabled or unsupported
}

// Optional addition to SettingsStore:
var hapticsEnabled: Bool  // default true
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Haptics feel annoying | M | M | Use light impact only; provide toggle |
| Over-firing on rapid taps | M | L | Debounce or use prepared generator once per action |
| Settings surface clutter | L | L | Consider tying to sound toggle if separate toggle unwanted |
