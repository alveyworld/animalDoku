## 1. Problem Statement

MVP puzzles are hand-authored JSON files bundled in the app, but there is no defined schema and no code to load them. Without a loader, the game cannot obtain a `Puzzle` to play, and puzzle authors have no contract to write against. A documented schema and a `PuzzleLoader` service unblock both content authoring and gameplay.

## 2. Goals

- Formalize the bundled puzzle JSON schema per the Formal Rules (`id`, `size`, `difficulty`, `regions` with `color`, `solution`, `initialPlacements`).
- Implement a `PuzzleLoader` service that decodes a bundled JSON file into a `Puzzle`.
- Validate structural invariants at load time (regions partition the board, region ids contiguous, `solution.length == size`).
- Surface clear, typed errors on malformed or missing puzzle files.

## 3. Non-Goals

- Rule validation of the solution (Rules 1–4) — that is the Validator (P2.4–P2.7); the loader checks structure only.
- Authoring actual puzzle content (P2.8).
- Puzzle selection UI (P4.8).
- Runtime puzzle generation (Future).

## 4. Personas & User Stories

- **As a puzzle author**, I want a documented JSON schema so that my files load without guesswork.
- **As a developer**, I want a `PuzzleLoader` that returns a `Puzzle` or a typed error so that the game can fail loudly on bad content.
- **As a player** (indirect), I want puzzles to load instantly so that the game starts without delay.

## 5. Functional Requirements

- **FR-1.** The system MUST define a JSON schema with top-level keys `id` (string), `size` (int), `difficulty` (string enum), `regions` (array), `solution` (array of positions), and `initialPlacements` (array of positions, `[]` for MVP).
- **FR-2.** Each region object MUST contain `id` (int), `color` (hex string), and `cells` (array of `{row, col}`).
- **FR-3.** `PuzzleLoader` MUST provide a method to load a puzzle by file name from the app bundle and return a decoded `Puzzle`.
- **FR-4.** `PuzzleLoader` MUST throw a typed error when the file is missing, JSON is malformed, or decoding fails.
- **FR-5.** `PuzzleLoader` MUST validate structural invariants and throw when: regions do not partition all `size × size` cells exactly once; region ids are not contiguous `0…size-1`; or `solution.count != size`.
- **FR-6.** `PuzzleLoader` SHOULD provide a method to list all bundled puzzle file names (for future puzzle select).
- **FR-7.** The loader MUST NOT import SwiftUI/UIKit.

## 6. Non-Functional Requirements

- **Performance** — Loading and structural validation of an 8×8 puzzle MUST complete in < 10 ms on device.
- **Reliability** — Malformed content MUST NOT crash; it MUST surface as a thrown error the caller can handle.
- **Maintainability** — Loader lives in `Services/`; schema documented in the puzzle folder README/comment.
- **Backward compatibility** — Unknown/extra JSON keys SHOULD be ignored so schema can grow additively.

## 7. Acceptance Criteria

- **AC-1.** *Given* a valid bundled puzzle JSON, *when* `PuzzleLoader.load(named:)` is called, *then* it returns a `Puzzle` with matching `id`, `size`, `regions`, and `solution`.
- **AC-2.** *Given* a puzzle where regions cover all 64 cells exactly once, *when* loaded, *then* structural validation passes.
- **AC-3.** *Given* a puzzle where a cell is claimed by two regions (or none), *when* loaded, *then* a partition error is thrown.
- **AC-4.** *Given* a puzzle where `solution.count != size`, *when* loaded, *then* a solution-length error is thrown.
- **AC-5.** *Given* a missing file name, *when* loaded, *then* a `fileNotFound` error is thrown (no crash).
- **AC-6.** *Given* malformed JSON, *when* loaded, *then* a decoding error is thrown.

## 8. Data Model

- Consumes `Puzzle`, `Region`, `Position`, `Difficulty` from P2.1 (no new domain types).
- Introduces a `PuzzleLoaderError` enum:

```swift
enum PuzzleLoaderError: Error, Equatable {
    case fileNotFound(String)
    case decodingFailed(String)
    case regionsDoNotPartition           // cells missing or double-claimed
    case nonContiguousRegionIds
    case solutionLengthMismatch(expected: Int, got: Int)
}
```

Reference JSON schema (per Formal Rules §Puzzle JSON Schema):

```json
{
  "id": "puzzle-001",
  "size": 8,
  "difficulty": "easy",
  "initialPlacements": [],
  "regions": [
    { "id": 0, "color": "#A8D8EA", "cells": [ { "row": 0, "col": 0 }, { "row": 0, "col": 1 } ] }
  ],
  "solution": [ { "row": 0, "col": 3 }, { "row": 1, "col": 7 } ]
}
```

## 9. API Surface

- No network APIs.
- Internal service surface:

```swift
struct PuzzleLoader {
    func load(named name: String, in bundle: Bundle = .main) throws -> Puzzle
    func availablePuzzleNames(in bundle: Bundle = .main) -> [String]
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Puzzle JSON not copied into bundle | M | H | Verify resource membership in build; add `fileNotFound` test |
| Region partition bug slips through | M | H | Structural validation with unit tests (AC-2, AC-3) |
| Schema drift from Formal Rules | L | M | Decode against the spec example; keep keys verbatim |

## 15. Rollout Plan

- No feature flag. Merge when AC pass.
- Rollback: loader is self-contained; safe to revert before GameSession depends on it.

## 16. Test Plan

- **Unit** — Load valid fixture → assert fields (AC-1). Partition pass (AC-2) and fail (AC-3). Solution-length mismatch (AC-4). Missing file (AC-5). Malformed JSON (AC-6). Use small fixtures (e.g. 4×4) plus the real 8×8 from P2.8.
- **Integration** — Load the actual bundled `puzzle-001.json` and confirm no throw.
- **Manual** — Add a temporary call in app startup to confirm the puzzle loads on simulator.

## 17. Documentation & Training

- Document the schema and invariants in a comment or README beside `Resources/Puzzles/`.

## 18. Open Questions

- Should structural validation run at load time always, or only in DEBUG for performance? Default: always (cost is negligible for 8×8).
