## 1. Problem Statement

The MVP milestone requires at least one real, playable 8×8 puzzle. Without authored content, the loader and validator have nothing to run on and the game cannot be played end-to-end. This story hand-authors `puzzle-001.json`, ensures it loads via `PuzzleLoader`, and verifies it has exactly one valid solution using the completed `Validator`.

## 2. Goals

- Hand-author `Resources/Puzzles/puzzle-001.json` as a valid 8×8 puzzle with 8 colored regions and a solution.
- Ensure the file conforms to the schema and structural invariants (regions partition all 64 cells; ids `0…7`; `solution.count == 8`).
- Verify the solution satisfies Rules 1–4 via the `Validator` (`isValid` and `isComplete` both true).
- Verify the puzzle has a **single** valid solution (offline solver or exhaustive check).

## 3. Non-Goals

- Building a runtime puzzle generator (Future).
- Authoring the full 5–10 puzzle set (v1.0) — MVP needs one.
- Difficulty tuning beyond labeling `easy`.
- UI to select puzzles (P4.8).

## 4. Personas & User Stories

- **As a player**, I want a real, solvable puzzle so that I can actually play the game.
- **As a developer**, I want a bundled puzzle so that GameSession and the board can be exercised end-to-end.
- **As a puzzle author**, I want a verification harness so that I can confirm uniqueness before shipping.

## 5. Functional Requirements

- **FR-1.** The system MUST include `Resources/Puzzles/puzzle-001.json` with `size == 8`, `difficulty == "easy"`, and `initialPlacements == []`.
- **FR-2.** The puzzle MUST define exactly 8 regions with contiguous ids `0…7`, each with a unique hex `color`, that partition all 64 cells exactly once.
- **FR-3.** The `solution` MUST contain exactly 8 positions (one per row) and MUST satisfy Rules 1–4.
- **FR-4.** The puzzle MUST have exactly one valid solution (verified offline).
- **FR-5.** The file MUST load successfully via `PuzzleLoader.load(named:)` with no structural errors.
- **FR-6.** A verification test/utility MUST confirm `Validator.validate(solutionBoard)` returns `isValid == true` and `isComplete == true`.
- **FR-7.** Each region SHOULD have a distinct visual pattern index reserved for colorblind support (Formal Rules invariant 4); if the schema does not yet carry pattern index, this is tracked as an open item.

## 6. Non-Functional Requirements

- **Performance** — Puzzle loads in < 10 ms; uniqueness verification is offline and may be slower (author-time only).
- **Reliability** — Single-solution property MUST be verified, not assumed; store the verification method/result note alongside the puzzle.
- **Maintainability** — Keep JSON human-readable (pretty-printed, one region per block) so future authors can edit.
- **Accessibility** — Region colors SHOULD be chosen from an accessible palette; pattern support tracked for Phase 5.

## 7. Acceptance Criteria

- **AC-1.** *Given* `puzzle-001.json`, *when* loaded via `PuzzleLoader`, *then* it returns a `Puzzle` with `size == 8` and 8 regions and no structural error.
- **AC-2.** *Given* the puzzle's 8 regions, *when* their cells are unioned, *then* they cover all 64 cells exactly once (no gaps, no overlaps).
- **AC-3.** *Given* a board with animals placed at every `solution` position, *when* validated, *then* `isValid` and `isComplete` are both `true`.
- **AC-4.** *Given* an offline uniqueness check (backtracking solver over the region layout), *when* run, *then* exactly one solution is found and it equals `solution`.
- **AC-5.** *Given* `initialPlacements`, *when* inspected, *then* it is `[]` (MVP starts empty).

## 8. Data Model

- No new types. Produces a data artifact `Resources/Puzzles/puzzle-001.json` conforming to the P2.1 `Puzzle` schema.
- Author-time verification uses `Validator` (P2.7) and a small backtracking solver (test-only utility) to prove uniqueness.

Skeleton (regions/solution filled during authoring):

```json
{
  "id": "puzzle-001",
  "size": 8,
  "difficulty": "easy",
  "initialPlacements": [],
  "regions": [ { "id": 0, "color": "#A8D8EA", "cells": [ /* ... */ ] } /* ...7 more... */ ],
  "solution": [ /* 8 positions, one per row */ ]
}
```

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Puzzle has multiple solutions | M | H | Run backtracking solver; require exactly one solution (AC-4) |
| Regions don't partition board | M | H | Loader validation (P2.3) + AC-2 |
| Puzzle unsolvable by logic (needs guessing) | M | M | Author with deduction in mind; content guideline (Formal Rules §Content Verification) |
| Colors not colorblind-safe | L | M | Pick accessible palette; track pattern index for Phase 5 |

## 15. Rollout Plan

- No feature flag. Merge when AC pass and verification recorded.
- Rollback: content-only artifact; safe to revert (game simply has no puzzle until re-added).

## 16. Test Plan

- **Unit / integration** — Load `puzzle-001.json` through `PuzzleLoader` (AC-1). Partition coverage check (AC-2). Validate solution board → valid + complete (AC-3). Single-solution solver run asserting one solution equal to `solution` (AC-4). `initialPlacements` empty (AC-5). This overlaps with the puzzle-integrity tests in Phase 6.
- **Manual** — Hand-solve the puzzle once to sanity-check it is enjoyable and deduction-solvable.

## 17. Documentation & Training

- Record the authoring process and single-solution verification result in a comment/README beside `Resources/Puzzles/`.
