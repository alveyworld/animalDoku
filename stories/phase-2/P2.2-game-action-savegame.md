## 1. Problem Statement

The game needs a formal representation of every player move so that undo/redo can be implemented, and a serializable snapshot of a game so that in-progress state can be saved later. Without a `GameAction` type, the session has no history to reverse; without `SaveGame`, there is no persistence contract for v1.1. Defining both now keeps the session and persistence work unblocked.

## 2. Goals

- Define a `GameAction` enum covering every reversible player move: place, remove, toggle blocked, and hint.
- Ensure each action carries enough data to be reversed for undo/redo.
- Define a `SaveGame` struct that round-trips a full game snapshot via JSON.
- Keep both types SwiftUI-free so the Engine layer can use them in tests.

## 3. Non-Goals

- Implementing the undo/redo stacks or applying actions (P3.3/P3.4 in GameSession).
- Writing `SaveGame` to disk (P4.7 persistence — schema only here).
- Timer or hint execution logic (Phase 3).

## 4. Personas & User Stories

- **As a developer**, I want a typed action for each move so that undo/redo can reverse it deterministically.
- **As a developer**, I want a `Codable` `SaveGame` so that persistence in v1.1 needs no schema redesign.
- **As a player** (indirect), I want my last move to be undoable so that mistakes are cheap.

## 5. Functional Requirements

- **FR-1.** The system MUST define a `GameAction` enum with cases for `place`, `remove`, `toggleBlocked`, and `hint`.
- **FR-2.** Each `GameAction` case MUST carry the `Position` it affects, and where needed the prior `CellState`, so the action is reversible without recomputation.
- **FR-3.** The `hint` action MUST record that the placement originated from a hint (so `hintsUsed` accounting is correct on undo).
- **FR-4.** The system MUST define `SaveGame { puzzleId, elapsedSeconds, cells, hintsUsed, mistakes, completed }` conforming to `Codable`.
- **FR-5.** `SaveGame.cells` MUST be `[Cell]` so a full board can be reconstructed.
- **FR-6.** `GameAction` and `SaveGame` MUST NOT import SwiftUI or UIKit.
- **FR-7.** `GameAction` SHOULD conform to `Equatable` to simplify testing of history stacks.

## 6. Non-Functional Requirements

- **Performance** — Actions are lightweight value types; pushing/popping history is O(1).
- **Maintainability** — One type per file under `Models/`; case names align with the Formal Rules player-action list.
- **Backward compatibility** — `SaveGame` includes `mistakes` (deferred to v1.1) so the schema is stable when persistence ships.
- **Reliability** — Reversal MUST be lossless: applying then undoing an action returns the exact prior board state.

## 7. Acceptance Criteria

- **AC-1.** *Given* a `place` action recording the prior `empty` state, *when* undone, *then* the target cell returns to `empty`.
- **AC-2.** *Given* a `toggleBlocked` action on an empty cell, *when* undone, *then* the cell returns to `empty` (and vice versa for clearing a block).
- **AC-3.** *Given* a `hint` action, *when* inspected, *then* it is distinguishable from a manual `place` (so `hintsUsed` can be decremented on undo).
- **AC-4.** *Given* a `SaveGame` value, *when* encoded to JSON and decoded, *then* the result equals the original (field-by-field).
- **AC-5.** *Given* the Models module, *when* compiled, *then* neither type imports SwiftUI/UIKit.

## 8. Data Model

New types under `Models/` (per Formal Rules §Player Actions and §SaveGame):

```swift
enum GameAction: Equatable {
    case place(at: Position, previous: CellState)
    case remove(at: Position, previous: CellState)
    case toggleBlocked(at: Position, previous: CellState)
    case hint(at: Position, previous: CellState)
}

struct SaveGame: Codable {
    let puzzleId: String
    let elapsedSeconds: Int
    let cells: [Cell]
    let hintsUsed: Int
    let mistakes: Int      // always 0 in MVP; tracked from v1.1
    let completed: Bool
}
```

- `previous` captures the pre-action `CellState` so undo needs no re-derivation.
- `mistakes` is defined now but not populated in MVP (Formal Rules §Game State).

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Action lacks data to reverse | M | H | Store `previous` state in every case; cover with undo tests |
| Hint indistinguishable from place | M | M | Dedicated `hint` case; assert in AC-3 |
| `SaveGame` schema churn later | L | M | Include all v1.1 fields now (`mistakes`, `elapsedSeconds`) |

## 15. Rollout Plan

- No feature flag. Merge when AC pass.
- Rollback: additive types; safe to revert before GameSession consumes them.

## 16. Test Plan

- **Manual** — Confirm compile with no UI imports (AC-5).
