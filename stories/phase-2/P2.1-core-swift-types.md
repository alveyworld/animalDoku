## 1. Problem Statement

The game has no domain types, so no engine, loader, or UI code can be written. Every subsequent feature — validation, JSON loading, game session, and rendering — depends on a shared, well-defined set of value types. Without them, work cannot proceed and there is no contract to test against.

## 2. Goals

- Implement the core value types (`Position`, `CellState`, `Cell`, `Region`, `Puzzle`, `Difficulty`, `Theme`) exactly as specified in the Formal Rules data model.
- Make types `Codable`/`Equatable` where required so puzzle JSON can decode into them.
- Keep all types free of SwiftUI and UIKit dependencies so the Engine layer stays unit-testable.
- Provide a single source of truth for the board coordinate system (`row`, `col`).

## 3. Non-Goals

- Validation logic (P2.4–P2.7).
- Game session, actions, or persistence (P2.2).
- Loading puzzles from JSON (P2.3).
- Theme asset catalog or color rendering (Phase 4/5).

## 4. Personas & User Stories

- **As a developer**, I want strongly-typed board primitives so that engine and UI code share one vocabulary.
- **As a developer**, I want `Codable` puzzle types so that I can decode bundled JSON without hand-written parsing.
- **As a QA engineer**, I want `Equatable` types so that I can assert exact state in unit tests.

## 5. Functional Requirements

- **FR-1.** The system MUST define `Position { row: Int, col: Int }` conforming to `Codable, Equatable`.
- **FR-2.** The system MUST define `CellState` as a `String`-backed `Codable` enum with cases `empty`, `blocked`, `animal`.
- **FR-3.** The system MUST define `Cell { row: Int, col: Int, regionId: Int, var state: CellState }` conforming to `Codable, Equatable`.
- **FR-4.** The system MUST define `Region { id: Int, color: String, cells: [Position] }` conforming to `Codable, Equatable`, where `color` is a hex string (e.g. `#A8D8EA`).
- **FR-5.** The system MUST define `Difficulty` as a `String`-backed `Codable` enum with cases `easy`, `medium`, `hard`, `expert`.
- **FR-6.** The system MUST define `Puzzle { id: String, size: Int, regions: [Region], solution: [Position], difficulty: Difficulty, initialPlacements: [Position] }` conforming to `Codable, Identifiable`.
- **FR-7.** The system MUST define `Theme { id: String, name: String, animal: String, icon: String, primaryColor: String, accentColor: String }` conforming to `Identifiable`.
- **FR-8.** Types MUST NOT import SwiftUI or UIKit.

## 6. Non-Functional Requirements

- **Performance** — Value types only; zero heap allocation beyond arrays. Decoding a puzzle MUST be < 5 ms.
- **Maintainability** — One type per file under `Models/`; names match the Formal Rules spec verbatim.
- **Backward compatibility** — `initialPlacements` present now (always `[]` for MVP) so future givens require no schema change.
- **Accessibility** — N/A (no UI).
- **Internationalization** — `Theme.name` is display copy and MUST be externalizable later; not localized in this story.

## 7. Acceptance Criteria

- **AC-1.** *Given* the Models module, *when* the project compiles, *then* all seven types build with no SwiftUI import.
- **AC-2.** *Given* a sample puzzle JSON, *when* decoded via `JSONDecoder`, *then* a `Puzzle` is produced with matching `id`, `size`, `regions`, and `solution`.
- **AC-3.** *Given* a decoded `Puzzle`, *when* re-encoded and decoded again, *then* the round-tripped value is `Equatable`-equal to the original.
- **AC-4.** *Given* two `Position` values with equal `row` and `col`, *when* compared, *then* they are equal.
- **AC-5.** *Given* `CellState` raw values, *when* decoded from the strings `"empty"`, `"blocked"`, `"animal"`, *then* the correct cases are produced.

## 8. Data Model

New types under `Models/` (per Formal Rules §Data Model):

```swift
struct Position: Codable, Equatable { let row: Int; let col: Int }

enum CellState: String, Codable { case empty, blocked, animal }

struct Cell: Codable, Equatable {
    let row: Int; let col: Int; let regionId: Int; var state: CellState
}

struct Region: Codable, Equatable {
    let id: Int; let color: String; let cells: [Position]
}

enum Difficulty: String, Codable { case easy, medium, hard, expert }

struct Puzzle: Codable, Identifiable {
    let id: String
    let size: Int
    let regions: [Region]
    let solution: [Position]
    let difficulty: Difficulty
    let initialPlacements: [Position]
}

struct Theme: Identifiable {
    let id: String; let name: String; let animal: String
    let icon: String; let primaryColor: String; let accentColor: String
}
```

- No database; puzzles are decoded from bundled JSON.
- `Region.color` and `Theme.*Color` are hex strings; conversion to `Color` is deferred to the UI layer.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Type shape drifts from Formal Rules | M | H | Copy signatures verbatim; add round-trip test against spec JSON |
| Coupling to SwiftUI creeps in | L | M | Keep Models target free of UI imports; enforce in review |
| `initialPlacements` omitted, breaking future | L | M | Include field now, default `[]` |
